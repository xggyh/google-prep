Google 原创 G18. Median of RLE Arrays 两个 RLE 数组的中位数

类别：📐 DP / 数组 · 难度：Hard · 标记：🆕 Google 原创

📐 DP / 数组 Hard 🆕 Google 原创

题目概览

🏷️ 类别：DP / 数组 / 二分 | 难度：Hard | 标记：Google 原创 | 来源：L3/L4 编译 #8

题目

EN: Two arrays in run-length encoded (RLE) form, both sorted by value. RLE format: [(value, count), ...]. Find median of the merged sorted array, without decoding.

中文：两个 RLE 数组（按值排序），不解码情况下求合并后中位数。

🧠 整体思路

二分查找：类似 LC 4 思路。

总元素数 N1 + N2，中位数位置 mid = (N1 + N2) // 2。

对值二分：候选值 v 是某 (value, count) 中的 value。算 "在合并数组中 ≤ v 的元素数 = cnt(v)"。 - cnt(v) > mid → 答案 ≤ v。 - cnt(v) ≤ mid → 答案 > v。

复杂度：O((|A|+|B|) log V)。

🪜 思路拆解

Step 1 · 计算 N1, N2, mid 位置 Step 2 · 二分 value 范围 [min, max] Step 3 · count(v) = sum of count for entries with value <= v Step 4 · 调整二分边界

⌨️ 逐步实现

def median_rle(A, B):
    """A, B: list of (value, count). Sorted by value."""
    def total(arr):
        return sum(c for _, c in arr)
    n = total(A) + total(B)
    target = n // 2

    def count_le(arr, v):
        return sum(c for val, c in arr if val <= v)

    # 二分 value
    all_vals = sorted(set(v for v, _ in A) | set(v for v, _ in B))
    lo, hi = 0, len(all_vals) - 1
    while lo < hi:
        mid = (lo + hi) // 2
        v = all_vals[mid]
        if count_le(A, v) + count_le(B, v) > target:
            hi = mid
        else:
            lo = mid + 1
    return all_vals[lo]

讲解策略

  1. 类似 LC 4，但元素以 RLE 压缩。
  2. 关键 count_le(v) 在 RLE 上 O(|A| + |B|)。
  3. 复杂度 O((|A|+|B|) log V)。

常见坑

  • 偶数 n 中位数：取两个中间值平均（这里简化为下中位数）。
  • RLE 中 count = 0 的条目要过滤。
