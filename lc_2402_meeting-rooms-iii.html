LC 2402. Meeting Rooms III 会议室 III

类别：📚 堆 / 扫描线 · 难度：Hard · 标记：➕ 额外推荐

📚 堆 / 扫描线 Hard ➕ 额外推荐

题目概览

🏷️ 类别：堆 / 模拟 | 难度：Hard | 标记：➕

题目

EN: n meeting rooms numbered 0..n-1. Meetings meetings[i] = [start, end]. Allocation rules: - Each meeting kept its original duration. - Earliest available room (smallest index) is used. - If all busy, delay meeting; meeting starts as soon as some room frees, keeping original duration.

Return the room that held the most meetings (smallest index if tie).

中文：n 间会议室编号 0..n-1。每个会议保持原时长。优先用空闲房中编号最小的；都忙就等最早空的房间，保持原时长。返回开会次数最多的房间（同次取最小编号）。

澄清问题

  • EN: How to break ties when rooms free at the same time? / Can meetings overlap exactly at endpoints?
  • 中：多个房间同时空闲怎么选？端点重合怎么处理？

🧠 整体思路

双堆： - free_heap：空闲房间，按 index 最小堆。 - busy_heap：在用房间，按 (end_time, index) 最小堆。

核心循环：按会议 start 排序，依次处理： - 先把 busy_heap 里 end ≤ start 的房间还回 free_heap。 - 若 free 非空：分配最小 index，更新计数 + 加到 busy。 - 若 free 空：pop busy 最早结束的房间，meeting 的 end = old_end + duration。

复杂度：O(M log N + M log M)。

🪜 思路拆解

Step 1 · 初始化两个堆 + 计数数组 - free_heap = list(range(n)) + heapify。 - busy_heap = []。 - count = [0] * n。

Step 2 · 排序会议 - 按 start 升序。

Step 3 · 每个会议处理 - 把 busy 里 end ≤ start 的还给 free。 - 若 free 非空：分配最小 idx，count[idx]++，push 到 busy（end）。 - 若 free 空：pop busy 最早的 (end, idx)，count[idx]++，新 end = end + duration，push 回 busy。

Step 4 · 选 count 最大的 index（tie 取小）

⌨️ 逐步实现

import heapq

def mostBooked(n, meetings):
    meetings.sort(key=lambda x: x[0])
    free = list(range(n))
    heapq.heapify(free)
    busy = []                                    # (end_time, room_idx)
    count = [0] * n

    for start, end in meetings:
        duration = end - start
        # 把已经空出来的房间还给 free
        while busy and busy[0][0] <= start:
            _, idx = heapq.heappop(busy)
            heapq.heappush(free, idx)

        if free:
            idx = heapq.heappop(free)
            heapq.heappush(busy, (end, idx))
        else:
            old_end, idx = heapq.heappop(busy)
            heapq.heappush(busy, (old_end + duration, idx))

        count[idx] += 1

    return count.index(max(count))               # 第一个 max 即最小 idx

讲解策略

  1. 双堆建模清晰：free 按 index，busy 按 end。
  2. 关键 idiom：每个新会议先"释放"早结束的房间。
  3. 复杂度 O(M log N)。
  4. 这是 LC 253 的进阶。

常见坑

  • <= vs <：end == start 时房间是否可立即用，要问。
  • 延迟会议的 end 计算：old_end + duration，不是 old_end + (end - start) 但等价。
  • tie：count.index(max(count)) 自然返回第一个最大值的索引。
