LC 253. Meeting Rooms II 会议室 II

类别：📚 堆 / 扫描线 · 难度：Med · 标记：⭐ Google 直接对应

📚 堆 / 扫描线 Med ⭐ Google 直接对应

题目概览

🏷️ 类别：堆 / 扫描线 | 难度：Medium | 标记：⭐➕

题目

EN: Given an array of meeting time intervals intervals[i] = [start, end], return the minimum number of conference rooms required.

中文：会议时间区间数组，求至少需要多少间会议室。

澄清问题

  • EN: Is end exclusive (room free at end time)? / Sorted?
  • 中：结束时间是开区间吗（结束时刻就能立刻给下一个用）？已排序吗？

🧠 整体思路

两种解法： 1. min-heap：按 start 排序；堆维护"已开会议的 end 时间"；新会议来若堆顶 end ≤ 当前 start 就复用，否则新开。 2. 扫描线：把 (start, +1) 和 (end, -1) 排序后遍历，维护当前在用房间数的最大值。

Heap 法更直观，O(n log n)。

🪜 思路拆解（Heap 版）

Step 1 · 按 start 排序 - intervals.sort(key=lambda x: x[0])。

Step 2 · 初始化堆 - 第一个会议的 end 入堆。

Step 3 · 遍历后续会议 - 若 heap[0] <= cur.start：pop（释放该房间）。 - 把 cur.end push 入堆。 - 堆大小代表"当前同时在用的房间数"，最终堆大小 = 总需要数。

Step 4 · 返回堆大小

⌨️ 逐步实现

import heapq

def minMeetingRooms(intervals):
    if not intervals:
        return 0
    intervals.sort(key=lambda x: x[0])           # Step 1
    heap = [intervals[0][1]]                      # Step 2
    for start, end in intervals[1:]:
        if heap[0] <= start:                      # 最早结束的房间已空
            heapq.heappop(heap)
        heapq.heappush(heap, end)
    return len(heap)

扫描线版（备用）

def minMeetingRooms_sweep(intervals):
    events = []
    for s, e in intervals:
        events.append((s, 1))
        events.append((e, -1))
    events.sort(key=lambda x: (x[0], x[1]))       # -1 优先 (会议先结束再开始)
    cur = mx = 0
    for _, delta in events:
        cur += delta
        mx = max(mx, cur)
    return mx

讲解策略

  1. 两种解法都讲，主推 heap。
  2. 强调"按 start 排序"是前提。
  3. 复杂度 O(n log n)。
  4. 提扩展：LC 2402 Meeting Rooms III 是这题的进阶。

常见坑

  • <= vs <：先问清楚。
  • 扫描线 tie-break：相同时刻 end 应排在 start 前面（先释放再分配）。
  • 空输入：return 0。
