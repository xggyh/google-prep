LC 347. Top K Frequent Elements 前 K 个高频元素

类别：📚 堆 / 扫描线 · 难度：Med · 标记：➕ 额外推荐

📚 堆 / 扫描线 Med ➕ 额外推荐

题目概览

🏷️ 类别：堆 / 桶排序 | 难度：Medium | 标记：➕

题目

EN: Given an integer array nums and integer k, return the k most frequent elements.

中文：返回数组中出现频率前 k 高的元素。

澄清问题

  • EN: Order of output? / Are ties guaranteed unique? / k always ≤ unique elements?
  • 中：输出有顺序要求吗？同频如何打破平局？k 一定 ≤ 不同元素数吗？

🧠 整体思路

两种主流解法： 1. min-heap size k：O(n log k)。 2. 桶排序：把元素按频率分桶（桶下标 = 频率），从高到低收集。O(n)，最优。

桶排序为什么 O(n)：频率最大值不会超过 n，所以桶数 ≤ n。

🪜 思路拆解（桶排序版）

Step 1 · 统计频率 - Counter(nums)。

Step 2 · 建桶 - buckets = [[] for _ in range(n + 1)]。 - 把每个 (元素, 频率) 放到 buckets[频率]。

Step 3 · 从高频到低频收集 - 从 buckets[n] 倒序到 buckets[1]。 - 收集前 k 个就 break。

⌨️ 逐步实现

from collections import Counter

def topKFrequent(nums, k):
    n = len(nums)
    freq = Counter(nums)
    buckets = [[] for _ in range(n + 1)]
    for num, f in freq.items():
        buckets[f].append(num)
    result = []
    for f in range(n, 0, -1):
        for num in buckets[f]:
            result.append(num)
            if len(result) == k:
                return result
    return result

Heap 版（备用）

import heapq
from collections import Counter

def topKFrequent_heap(nums, k):
    freq = Counter(nums)
    return [x for x, _ in heapq.nlargest(k, freq.items(), key=lambda p: p[1])]

讲解策略

  1. 桶排序 vs heap 的 trade-off。
  2. 桶排序适用于"频率有上界"的场景（这里上界是 n）。
  3. 复杂度 O(n)。

常见坑

  • 桶大小：n + 1 不能写 n（频率可以等于 n）。
  • tie：相同频率的元素任意顺序，先问清楚。
