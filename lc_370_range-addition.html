LC 370. Range Addition 区间加法

类别：📚 堆 / 扫描线 · 难度：Med · 标记：➕ 额外推荐

📚 堆 / 扫描线 Med ➕ 额外推荐

题目概览

🏷️ 类别：堆 / 扫描线 / 差分数组 | 难度：Medium | 标记：➕

题目

EN: Length n array initialized 0. Given updates[i] = [startIdx, endIdx, inc]. For each update, add inc to arr[startIdx..endIdx] (both inclusive). Return final array.

中文：长 n 数组全 0。一组 [l, r, inc] 更新，把 arr[l..r] 都加 inc。返回最终数组。

澄清问题

  • EN: endIdx inclusive? / Number of updates?
  • 中：r 是闭区间吗？更新次数？

🧠 整体思路

差分数组：把每次 O(n) 的区间更新降到 O(1)。最后一次前缀和还原。

核心 idiom：

diff[l] += inc
diff[r+1] -= inc

最后 arr[i] = sum(diff[0..i]) 就是结果。

复杂度：O(N + K)，K 是更新次数。

🪜 思路拆解

Step 1 · 建差分数组 - diff = [0] * (n + 1)（多一个位置避免越界）。

Step 2 · 每个 update 做差分 - diff[l] += inc - diff[r+1] -= inc

Step 3 · 前缀和还原 - 遍历一遍累加。

⌨️ 逐步实现

def getModifiedArray(length, updates):
    diff = [0] * (length + 1)
    for l, r, inc in updates:
        diff[l] += inc
        diff[r + 1] -= inc
    arr = [0] * length
    cur = 0
    for i in range(length):
        cur += diff[i]
        arr[i] = cur
    return arr

讲解策略

  1. 差分数组是经典 O(1) 区间更新模板。
  2. 多分配 1 个位置避免 r+1 越界。
  3. 复杂度 O(N + K) 比朴素 O(N·K) 好很多。

常见坑

  • r+1 越界：必须分配 length+1 大小。
  • 闭区间 vs 半开：要先问。
  • 多次累加问题：单次 update 只动两个位置，重叠 update 也只是叠加这两个位置。
