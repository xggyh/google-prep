09. Game of Life（LC 289）

来源：SDE NG VO R2 题1 · LC: LC 289 ✅ 高置信 · 难度: Med

In-place Simulation ✅ 高置信 Med

题目概览

来源：Google SDE NG VO R2 第一题 原描述：李扣尔把酒（LC 289）。Follow-up：in-place modify。

题目

EN: Given an m x n board of cells, each 0 (dead) or 1 (alive). Update to the next generation according to Conway's rules: 1. Live cell with <2 live neighbors → dies (underpopulation). 2. Live cell with 2 or 3 → lives. 3. Live cell with >3 → dies (overpopulation). 4. Dead cell with exactly 3 → becomes alive. Follow-up: modify in-place (O(1) extra space).

中文：m×n 棋盘，1 活 0 死。按 Conway 生命游戏规则更新到下一代。Follow-up：原地修改。

澄清问题

  • EN: Are all 8 neighbors considered (including diagonals)? / Update synchronously (all cells based on current state) or asynchronously? / What about the infinite board edge case in real Conway — here it's fixed board? / Multiple steps or just one?
  • 中：邻居包括对角线吗（应该是 8 邻）？所有格子同时更新（基于上一代）？真实 Conway 是无限棋盘，这里是固定边界？只做一步还是多步？

Python 实现（in-place 用状态编码）

def gameOfLife(board):
    if not board or not board[0]:
        return
    m, n = len(board), len(board[0])
    # 编码：
    # 0 -> 0 (dead, stays dead)
    # 1 -> 1 (live, stays live)
    # 2 -> live, becomes dead next gen
    # 3 -> dead, becomes live next gen

    for i in range(m):
        for j in range(n):
            live = 0
            for di in (-1, 0, 1):
                for dj in (-1, 0, 1):
                    if di == 0 and dj == 0:
                        continue
                    ni, nj = i + di, j + dj
                    if 0 <= ni < m and 0 <= nj < n and board[ni][nj] in (1, 2):
                        live += 1
            if board[i][j] == 1 and (live < 2 or live > 3):
                board[i][j] = 2
            elif board[i][j] == 0 and live == 3:
                board[i][j] = 3

    for i in range(m):
        for j in range(n):
            board[i][j] = 1 if board[i][j] in (1, 3) else 0

讲解策略

  1. 先写 O(mn) 额外空间版：复制一份再修改，最清晰。
  2. Follow-up 抛出"in-place"：解释为什么直接改会乱 — 当前格变了，邻居算邻居数时看到的就是错的。
  3. 状态编码技巧：用 2 和 3 区分"曾经活/死、新一代死/活"，这是最常见的 O(1) 额外空间解法。
  4. 第二次遍历恢复：把 2 和 3 还原成 0 和 1。
  5. 更进一步 follow-up：如果棋盘无限大怎么办？答：只存活细胞的坐标 set，对每个活细胞的邻居计数 → HashLife / sparse representation。

常见坑

  • 同步更新理解错：每个格子的新状态必须基于所有格子的原始状态，不能边算边改。
  • 位运算编码反而出错：状态编码用整数 0/1/2/3 比 bitmask 清晰。
  • 边界判断 0 <= ni < m：少写一个 = 直接越界。
  • "无限大棋盘"的 follow-up：很多人没准备，提前练一下。
