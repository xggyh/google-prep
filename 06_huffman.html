Q06. Huffman 编码树生成（LC 1167 思想）

来源：L4 SWE 2026 R4 / L6 R2 · LC: LC 1167 变体 🟡 变体 · 难度: Med

Greedy Heap Tree 🟡 变体 Med

题目概览

来源：Google L4 SWE 2026 R4（Variation of Huffman encoding）+ Google L6 Onsite R2（生成哈夫曼树）

题目

EN: Given a list of characters with frequencies (or just a string), build a Huffman tree and produce a prefix code for each character. Optimize the total encoded length.

中文：给一组字符及其频率（或一个字符串），生成 Huffman 树，输出每个字符的前缀编码。目标：总编码长度最短。

澄清问题

  • EN: Input format — frequency map or raw string? / Need both the tree and the codes, or just the codes? / Tie-breaking when two nodes have same frequency? / Should encoding be deterministic across runs? / What if only one unique character?
  • 中：输入是字符串还是频率表？要返回树本身还是编码 map？频率相同时怎么打破平局（影响树形，但不影响最短长度）？是否要求确定性输出？只有一种字符的边界情况怎么办？

🧠 整体思路

Huffman 是经典贪心：每次合并频率最小的两个节点，构造一棵二叉树，叶子是字符，路径（0/1）就是编码。

为什么贪心是最优的：让频率高的字符靠近根（编码短），频率低的远离根（编码长），加权路径长度最小。

数据结构选 min-heap：每次拿"频率最小的两个"是 O(log n)；如果用 sorted list 每次插入 O(n)，总 O(n²)。

树节点定义： - 叶子：有 char 和 freq，无子节点。 - 内部：无 char，freq = 左子 + 右子，有 left/right。

编码生成：DFS 这棵树，左走 '0' 右走 '1'，到叶子时把 prefix 存到 codes[char]。

🪜 思路拆解

Step 1 · 统计频率 - 做什么：freq = Counter(text) 或直接给的频率表。 - 边界：只有 1 种字符时直接返回 {ch: "0"}，不然下面建树的 while 循环会无限等。

Step 2 · 定义 Node 类（带 tie-breaker） - 字段：freq、char、left、right。 - 关键：用 @dataclass(order=True) + 一个 tie 字段。 - 为什么 tie：堆里两个 Node 频率相同时，Python 会继续比较下一个字段；如果是 Node 自身，会抛 TypeError: '<' not supported。加一个递增计数器解决。

Step 3 · 初始化堆 - 把每个字符做成叶子 Node，全部 push 进 heap。

Step 4 · 合并循环 - 当堆大小 > 1 时： - pop 出最小的两个 a, b。 - 创建新节点 merged(freq = a.freq + b.freq, left = a, right = b)。 - push 回堆。 - 循环结束堆里只剩一个节点 — 这就是根。

Step 5 · 递归生成编码 - DFS 树：到叶子 (char != None) 就把当前 prefix 存到 codes[char]。 - 内部节点：左走 prefix + "0"，右走 prefix + "1"。

Step 6 · Edge cases - 空输入：返回 {}。 - 单字符：上面 Step 1 已处理。 - 极度偏斜的树（一边长链）：递归深度可能很大，用 sys.setrecursionlimit 或改迭代。

from typing import Optional

@dataclass(order=True)
class Node:

    char: Optional[str] = field(default=None, compare=False)

def build_huffman(text: str) -> dict:
    freq = Counter(text)
    if len(freq) == 1:                    # Step 6: 单字符 edge case
        ch, _ = next(iter(freq.items()))
        return {ch: "0"}

    heap = []
    for i, (ch, f) in enumerate(freq.items()):
        heapq.heappush(heap, Node(f, i, ch))
    tie = len(freq)                       # 后续合并节点的 tie 起点

Step 4 · 合并循环

    while len(heap) > 1:
        a = heapq.heappop(heap)
        b = heapq.heappop(heap)
        merged = Node(a.freq + b.freq, tie, None, a, b)
        tie += 1
        heapq.heappush(heap, merged)
    root = heap[0]

    codes = {}
    def assign(node, prefix):

            codes[node.char] = prefix
            return
        assign(node.left,  prefix + "0")
        assign(node.right, prefix + "1")

    assign(root, "")
    return codes

Python 完整实现

import heapq
from collections import Counter
from dataclasses import dataclass, field
from typing import Optional


@dataclass(order=True)
class Node:
    freq: int
    tie: int = field(compare=True)
    char: Optional[str] = field(default=None, compare=False)
    left: Optional["Node"] = field(default=None, compare=False)
    right: Optional["Node"] = field(default=None, compare=False)


def build_huffman(text: str) -> dict:
    freq = Counter(text)
    if len(freq) == 1:
        ch, _ = next(iter(freq.items()))
        return {ch: "0"}

    heap = []
    for i, (ch, f) in enumerate(freq.items()):
        heapq.heappush(heap, Node(f, i, ch))

    tie = len(freq)
    while len(heap) > 1:
        a = heapq.heappop(heap)
        b = heapq.heappop(heap)
        merged = Node(a.freq + b.freq, tie, None, a, b)
        tie += 1
        heapq.heappush(heap, merged)

    codes = {}
    _assign(heap[0], "", codes)
    return codes


def _assign(node, prefix, codes):
    if node.char is not None:
        codes[node.char] = prefix
        return
    _assign(node.left, prefix + "0", codes)
    _assign(node.right, prefix + "1", codes)

讲解策略

  1. 先解释 Huffman 是什么：贪心算法，每次合并频率最小的两个节点 — 让面试官确认你理解算法。
  2. 画图走一遍：用 5 个字符的小例子，从频率表逐步合并成树。
  3. 数据结构选 min-heap：解释为什么不能用 sorted list（每次插入 O(n)，总 O(n²)）。
  4. 复杂度：O(n log n)，n 是 unique chars。
  5. Edge cases：单字符（树退化为单点，约定编码"0"）；空输入。
  6. 进阶可以聊：Huffman vs 算术编码 / Range coding，工程中 Huffman 配 LZ77 = DEFLATE。

常见坑

  • tie-breaker 没处理：Python heapq 在 freq 相等时会比较第二个字段，如果是 Node 对象不支持比较会抛 TypeError。必须加 tie 计数器。
  • 单字符退化：直接跑算法会卡在 while len(heap) > 1，要单独处理。
  • 递归赋码爆栈：极偏斜的输入会让树深度接近 n，要么改迭代要么 sys.setrecursionlimit。
  • L6 评论提到"考的时间 25 分钟，要不停 clarify"：上来别埋头写代码，先快速跟面试官敲定输入输出和 Node 结构。
