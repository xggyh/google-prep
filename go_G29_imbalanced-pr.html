Google 原创 G29. Imbalanced Precision-Recall 不平衡数据 PR 曲线

类别：🧮 数学 / 概率 / DS · 难度：Hard · 标记：🆕 Google 原创

🧮 数学 / 概率 / DS Hard 🆕 Google 原创

题目概览

🏷️ 类别：数学 / 概率 / DS | 难度：Hard | 标记：Google 原创 | 来源：L3/L4 编译 #39

题目

EN: Compute evaluation metrics + PR curve for highly imbalanced dataset (e.g., 5% positives).

中文：5% 正样本的极不平衡数据集，计算 PR 曲线和评估指标。

🧠 整体思路

步骤： 1. 把 predictions（score）按降序排。 2. 从高 threshold 到低，每步更新 TP/FP/FN。 3. 计算 precision = TP/(TP+FP)，recall = TP/(TP+FN)。 4. 输出 (recall, precision) 序列即 PR 曲线。

不平衡问题：accuracy 没用（猜全负 95% 准），重点看 Precision、Recall、F1、PR-AUC。

🪜 思路拆解

Step 1 · 按 score 降序排序 (score, label) Step 2 · 累加 TP, FP Step 3 · 每个阈值点输出 (recall, precision)

⌨️ 逐步实现

def pr_curve(scores, labels):
    """scores: list of model output scores
       labels: list of 0/1 true labels"""
    paired = sorted(zip(scores, labels), key=lambda x: -x[0])
    total_pos = sum(labels)
    if total_pos == 0:
        return []
    tp = fp = 0
    pr_points = []
    for s, l in paired:
        if l == 1:
            tp += 1
        else:
            fp += 1
        prec = tp / (tp + fp)
        rec = tp / total_pos
        pr_points.append((rec, prec))
    return pr_points

讲解策略

  1. 按 score 排序后扫一遍。
  2. 不平衡时 PR-AUC 比 ROC-AUC 更有意义。
  3. 提 F1 = 2·P·R/(P+R)。

常见坑

  • accuracy 在不平衡下误导。
  • 0 阈值时所有正类都 recall。
  • precision 在 (TP+FP=0) 时未定义，按 1 处理。
