Google 原创 G28. Invoice Reconciliation 发票对账

类别：🧮 数学 / 概率 / DS · 难度：Easy · 标记：🆕 Google 原创

🧮 数学 / 概率 / DS Easy 🆕 Google 原创

题目概览

🏷️ 类别：数学 / 概率 / DS | 难度：Easy | 标记：Google 原创 | 来源：L3/L4 编译 #38

题目

EN: Match invoice records to payments based on memo text or exact amount.

中文：把发票和付款记录配对，依据 memo 文本或金额完全匹配。

🧠 整体思路

两阶段匹配： 1. 用 amount 建 HashMap（amount → list of invoices）。 2. 对每个 payment，先按 amount 查；amount 多个匹配时用 memo 文本相似度（exact match / Jaccard / fuzzy）打破平局。

🪜 思路拆解

Step 1 · 按 amount 索引 invoices Step 2 · 对每个 payment： - 找 amount 相同的 invoices - 0 个 → 未匹配 - 1 个 → 直接配对 - 多个 → 按 memo 相似度排序，取最匹配

⌨️ 逐步实现

from collections import defaultdict

def reconcile(invoices, payments):
    """invoices: list of (id, amount, memo)
       payments: list of (id, amount, memo)"""
    by_amount = defaultdict(list)
    for inv in invoices:
        by_amount[inv[1]].append(inv)

    pairs = []
    for pay in payments:
        candidates = by_amount[pay[1]]
        if not candidates:
            continue
        if len(candidates) == 1:
            pairs.append((pay[0], candidates[0][0]))
            by_amount[pay[1]].remove(candidates[0])
        else:
            # memo 相似度
            best = max(candidates, key=lambda c: _similarity(pay[2], c[2]))
            pairs.append((pay[0], best[0]))
            by_amount[pay[1]].remove(best)
    return pairs

def _similarity(a, b):
    sa, sb = set(a.split()), set(b.split())
    return len(sa & sb) / max(len(sa | sb), 1)

讲解策略

  1. 两层 fallback：金额 → 文本相似度。
  2. Jaccard / cosine / edit distance 可选。
  3. Follow-up：金额近似匹配（允许 ±1 美元）。

常见坑

  • 一对多 / 多对一：要先 clarify 是否允许。
  • 多个相似度相同：先到先服务 vs 全部返回。
