LC 2655. Find Maximal Uncovered Ranges 寻找最大长度的未覆盖区间

类别：📚 堆 / 扫描线 · 难度：Med · 标记：➕ 额外推荐

📚 堆 / 扫描线 Med ➕ 额外推荐

题目概览

🏷️ 类别：扫描线 / 区间合并 | 难度：Medium | 标记：➕

题目

EN: Given integer n and intervals ranges[i] = [start, end] (inclusive). Return all maximal uncovered ranges in [0, n-1] (sorted by start).

中文：给定 n 和一组闭区间 ranges，求 [0, n-1] 中所有最长的未被覆盖区间，按 start 升序返回。

澄清问题

  • EN: Can ranges overlap or be unsorted? / Inclusive endpoints?
  • 中：区间可以重叠 / 乱序吗？闭区间还是开区间？

🧠 整体思路

两步： 1. 合并重叠区间：按 start 排序，遍历合并。 2. 取补集：在 [0, n-1] 里找合并后区间的"间隙"。

复杂度：O(k log k)，k 是 ranges 数量。

🪜 思路拆解

Step 1 · 边界 - ranges 为空 → 整个 [0, n-1] 都是未覆盖。

Step 2 · 按 start 排序

Step 3 · 合并重叠区间 - 维护当前 merged 区间 (cur_s, cur_e)。 - 新区间 start ≤ cur_e + 1：扩展 cur_e = max(cur_e, end)（闭区间用 +1 判相邻）。 - 否则：把 (cur_s, cur_e) 收入 result，重置。

Step 4 · 取补集 - 在 merged 区间之前 / 之间 / 之后找未覆盖段。

⌨️ 逐步实现

def findMaximalUncoveredRanges(n, ranges):
    if not ranges:
        return [[0, n - 1]] if n > 0 else []

    ranges.sort(key=lambda x: x[0])
    # Step 3: 合并
    merged = [list(ranges[0])]
    for s, e in ranges[1:]:
        if s <= merged[-1][1] + 1:
            merged[-1][1] = max(merged[-1][1], e)
        else:
            merged.append([s, e])

    # Step 4: 取补
    result = []
    prev_end = -1
    for s, e in merged:
        if s > prev_end + 1:
            result.append([prev_end + 1, s - 1])
        prev_end = e
    if prev_end < n - 1:
        result.append([prev_end + 1, n - 1])
    return result

讲解策略

  1. 拆两步：合并 → 取补。
  2. 经典区间合并模板。
  3. 闭区间 vs 开区间影响 <= merged[-1][1] + 1 的 +1。
  4. 复杂度 O(k log k)。

常见坑

  • 闭区间 +1：[1, 3] 和 [4, 5] 应该合并（相邻），所以判 s <= e + 1。
  • 边界 [0, n-1]：补集的左右边界要包含端点。
  • 空 ranges：return 整段。
