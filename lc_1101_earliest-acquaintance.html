LC 1101. The Earliest Moment When Everyone Become Friends 彼此熟识的最早时间

类别：🗺️ 图 / BFS / DFS · 难度：Med · 标记：⭐ Google 直接对应

🗺️ 图 / BFS / DFS Med ⭐ Google 直接对应

题目概览

🏷️ 类别：图 / Union-Find | 难度：Medium | 标记：⭐

题目

EN: Given logs[i] = [timestamp, x, y] representing a friendship event. Friendship is transitive. Return the earliest timestamp when all n people are in one friend group, or -1.

中文：日志数组 [时间戳, x, y] 表示 x 和 y 成为朋友（关系传递）。返回所有 n 人成为一个朋友圈的最早时间，找不到返回 -1。

澄清问题

  • EN: Are logs sorted? / Symmetric friendship? / Can the same pair appear twice?
  • 中：是否已排序？关系对称吗？同一对会出现多次吗？

🧠 整体思路

Union-Find 模板题：按时间排序事件，每次 union；维护连通分量计数，到 1 就返回。

关键 invariant：友谊单调（只增不减）→ DSU 完美适用。

复杂度：O(E log E + E·α(N))。

🪜 思路拆解

Step 1 · 按 timestamp 排序 - 找最早时间必须按时间顺序处理。

Step 2 · UF 三件套：parent / rank / components - parent[i] = i，rank[i] = 0，components = n。

Step 3 · find with path compression - 沿 parent 爬到根；同时 path halving。

Step 4 · union by rank - 同根 → return False。 - 矮树挂高树下；rank 相等时新根 rank+1。 - 合并成功 → components -= 1。

Step 5 · 主循环 + 早停 - union 后检查 components == 1 → 返回 t。

⌨️ 逐步实现

Step 2 · UF 状态

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.components = n

Step 3 · find + 路径压缩

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

Step 4 · union by rank

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        self.components -= 1
        return True

Step 1 + 5 · 主函数

def earliestAcq(logs, n):
    logs.sort(key=lambda x: x[0])
    uf = UnionFind(n)
    for t, a, b in logs:
        uf.union(a, b)
        if uf.components == 1:
            return t
    return -1

讲解策略

  1. 抽象成"图连通分量"问题。
  2. 选 DSU 而非 BFS（DSU O(α) vs BFS O(V+E) per check）。
  3. 路径压缩 + 按秩合并，复杂度近似 O(1) per op。
  4. 早停优化：components = 1 立即返回。

常见坑

  • logs 未排序：题目可能给已排序，但要先问。
  • union 同根没 return False：components 计数会出错。
  • 递归 find 在大 N 爆栈：用迭代。
