15. Quick Select（LC 215 Kth Largest）

来源：Cloud SDE Phone · LC: LC 215 ✅ 高置信 · 难度: Med

Quickselect Random Pivot Partition ✅ 高置信 Med

题目概览

来源：Google Cloud SDE Phone Coding 原描述：问 quick select，讨论不同方法、pivot 怎么选，最后随便选一个 implement。

题目

EN: Given an unsorted array and integer k, return the k-th largest element. Discuss approaches and pivot strategies, then implement one.

中文：无序数组 + k，返回第 k 大元素。讨论方法和 pivot 选取，然后实现。

澄清问题

  • EN: k is 1-indexed (kth largest) or 0-indexed? / Are duplicates allowed? / Modify input in place ok? / Average vs worst case requirement? / Memory constraint?
  • 中：k 是 1-indexed 还是 0-indexed？有重复元素吗？是否允许原地修改？要求平均还是最坏复杂度？是否有内存限制（流式处理？）？

方法对比（要主动讲）

方法 Time Space 备注
Sort + index O(n log n) O(1) or O(n) 简单，可接受
Max-heap pop k 次 O(n + k log n) O(n) k 小时好
Min-heap size k O(n log k) O(k) streaming 友好
Quick select 随机 pivot O(n) avg, O(n²) worst O(log n) stack 期望最优
Median of medians O(n) worst O(log n) 理论最优，常数大

Python 实现（随机 pivot Quick Select）

import random


def quick_select_kth_largest(nums, k):
    """k is 1-indexed: k=1 returns max."""
    nums = nums[:]  # 不污染输入
    target = len(nums) - k  # 转成第 (n-k) 小

    def select(lo, hi):
        if lo == hi:
            return nums[lo]
        pivot_idx = random.randint(lo, hi)
        pivot = nums[pivot_idx]
        # 三路划分（处理重复）
        lt, gt = lo, hi
        i = lo
        while i <= gt:
            if nums[i] < pivot:
                nums[lt], nums[i] = nums[i], nums[lt]
                lt += 1
                i += 1
            elif nums[i] > pivot:
                nums[gt], nums[i] = nums[i], nums[gt]
                gt -= 1
            else:
                i += 1
        if target < lt:
            return select(lo, lt - 1)
        elif target > gt:
            return select(gt + 1, hi)
        else:
            return pivot

    return select(0, len(nums) - 1)

讲解策略

  1. 先列出方法表（不写代码），让面试官知道你了解全景。
  2. 解释 pivot 选取： - 永远取第一个 → 已排序输入退化到 O(n²)。 - 随机 → 期望 O(n)，工程默认选择。 - Median of medians → 最坏 O(n)，常数大。
  3. 三路 partition：解释为什么处理重复元素时三路比 Lomuto/Hoare 更稳。
  4. 复杂度证明：每次期望减半 → T(n) = T(n/2) + O(n) = O(n)。
  5. 实操选随机 pivot：写代码，dry run 一次。

常见坑

  • 递归到一个空区间：lo > hi 处理。
  • Lomuto partition + 重复元素：会一直把 pivot 推到一边，O(n²)。
  • k = 1 vs k = 0 边界：在 docstring 明确。
  • 没用 random.seed：面试时不会要求，但要主动说"实际 prod 不能用"。
