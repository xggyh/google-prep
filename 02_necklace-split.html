02. 项链切割平分 D/R（无直接 LC，本题 + 滑窗）

来源：L4 AI/ML 四轮 R4 · LC: 无直接 LC ❓ 无对应 · 难度: Hard

Sliding Window Greedy Custom ❓ 无对应 Hard

题目概览

来源：Google L4 AI/ML 四轮全套面经 R4 原描述：由 D 和 R 组成的字符串（#D = #R），切开后分给两个人，保证两人的 D 和 R 都相等。关键：at most 2 cuts + sliding window。

题目

EN: A circular string of 'D' and 'R' with equal count. Split it by at most 2 cuts so that two people each receive segments with equal counts of D and R. Return the cut positions (or just True/False for feasibility).

中文：一个由 D 和 R 组成的字符串，D 和 R 数量相等。最多切 2 刀，分给两人，使两人各自的 D 数和 R 数都相等。返回切的位置（或是否可行）。

澄清问题

  • EN: Is the string circular or linear? / "At most 2 cuts" — does 1 cut count? / Is it guaranteed #D == #R? / Each person must get a non-empty segment? / Can the two segments interleave (2 cuts producing 3 pieces, one person takes the middle one)?
  • 中：字符串是环形还是线性？2 刀指的是最多 2 刀还是恰好 2 刀？是否保证 D 和 R 数量相等？两人是否可以拿不连续的段（如 2 刀切 3 段，一人拿中间，一人拿两边）？

关键观察

  1. 1 刀可解的情况：从某个位置切开，左右两段 D/R 数量都各占一半 → 滑动起点（如果环形）或简单前缀和（如果线性）。
  2. 2 刀必解情况：当 1 刀不行时，2 刀一定能行 — 切出中间一段长度 = n/2 且其中 D 和 R 各占总数一半即可。这转化为长度为 n/2 的滑动窗口找一个窗口里 D 数 = R 数 = n/4。

Python 实现

def split_necklace(s):
    n = len(s)
    total_d = s.count('D')
    total_r = n - total_d
    if total_d != total_r:
        return None  # 题目保证不会发生，但 defensive
    half_d, half_r = total_d // 2, total_r // 2

    # 1 cut: 前缀的 D == half_d 且 R == half_r
    cd = cr = 0
    for i in range(n - 1):
        if s[i] == 'D':
            cd += 1
        else:
            cr += 1
        if cd == half_d and cr == half_r:
            return [i + 1]  # 在 i 和 i+1 之间切

    # 2 cuts: 长度 n/2 的窗口内 D 数 = half_d
    if n % 2 != 0:
        return None
    w = n // 2
    cd = sum(1 for c in s[:w] if c == 'D')
    if cd == half_d:
        return [0, w]
    for i in range(w, n):
        cd += (s[i] == 'D') - (s[i - w] == 'D')
        if cd == half_d:
            return [i - w + 1, i + 1]
    return None

讲解策略

  1. 先做 brute force：枚举所有 1-cut 和 2-cut 组合 — O(n²)，要明确说"这是 baseline，我会优化"。
  2. 关键 insight：必然存在解 ⇒ "如果 1 刀切不出来，那 2 刀一定能" 这个 claim 要证明（前缀和的奇偶性 + 中间值定理类似论证）。
  3. 滑动窗口降到 O(n)：解释固定长度 n/2 的窗口怎么 O(1) 维护 D 计数。
  4. 复杂度对比：从 O(n²) 到 O(n)，space O(1)。
  5. Edge case 主动列：n 奇数？全 D 全 R？只有 2 个字符？

常见坑

  • 以为 1 刀就够：很多情况下需要 2 刀，必须想清楚什么时候必须 2 刀。
  • 环形 vs 线性混淆：必须先问。
  • 没意识到 2 刀就是滑窗：被"切两刀"的语言迷惑，想 DP 或回溯，浪费时间。
  • 数学证明跳过：面试官常会问"为什么 2 刀够"，没准备会卡。
