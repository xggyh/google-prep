Q17. 以 target 为前缀的子串计数 + 最小包含子串

来源：Cloud SDE Coding 2 · LC: 无直接 LC ❓ 无对应 · 难度: Med

KMP Z-Algorithm String ❓ 无对应 Med

题目概览

来源：Google Cloud SDE Coding 2 原描述：输入 string 和 target，求有多少个以 target 为前缀的 substring。Follow-up：最小包含 target 的 substring 长度。

题目

EN: - Q1: Given s and target, count the number of substrings of s that start with target as a prefix. - Q2 (Follow-up): Find the length of the shortest substring of s that contains target (as substring, not prefix).

中文： - Q1：字符串 s 和 target，求 s 中有多少子串以 target 为前缀。 - Q2：求 s 中包含 target 的最短子串长度。

澄清问题（面经原文）

  • EN: Is len(s) >= len(target) guaranteed? / If len(s) < len(target), return 0, -1, null, or empty? / Are these case-sensitive? / What if target is empty?
  • 中：保证 |s| >= |target| 吗？如果 |s| < |target|，输出什么（0 / -1 / null / 空）？大小写敏感吗？target 空字符串怎么办？

🧠 整体思路

两段题，难点是 Q2 的歧义。

Q1（计数）：找 target 在 s 中所有的出现位置，每个出现贡献 (n - i - m + 1) 个 substring（i 是出现位置）— 这个 substring 可以是 target 本身，也可以包含 target 后面的若干字符。

Q2（最短包含 target）：先澄清歧义！"包含 target" 有两种解读： - 连续子串：target 作为 substring → 答案就是 len(target)（trivial）。 - 字符集合：target 的所有字符（顺序无所谓）→ 这是 LC 76 Minimum Window Substring，滑动窗口。

这一步不澄清就写错算法。

找出现位置用 KMP：O(n+m)，比朴素 O(nm) 好；但面试 30 分钟内手写 KMP 容易出 bug，Python 的 str.find 配 while 循环也能 O(nm) 跑通，更保险。

🪜 思路拆解

Step 1 · 澄清 Q2 含义 - 直接问面试官："包含 target 是指连续子串还是字符集合？" - 等回答再决定写哪个算法。

Step 2 · Q1 算法 - 找所有 target 在 s 中的起点 occ。 - 累加 sum(n - i - m + 1 for i in occ)。 - 边界：occ 空 → 返回 0；target 空 → 所有非空子串数（罕见，先问）。

Step 3 · KMP 失败函数 + 主匹配 - lps[i] = target[:i+1] 的最长 proper prefix that is also a suffix 的长度。 - 主匹配：用 lps 跳过已知匹配前缀，O(n)。

Step 4 · Q2 (连续子串解读) 答案 - target 存在 s 中 → 答案就是 len(target)。 - 不存在 → 返回 -1 或 0。

Step 5 · Q2 (multiset 解读) 滑动窗口（LC 76 套路） - 窗口里 char count 满足 target 的需求时收缩，记录最短长度。 - 这是另一题，本面经题大概率是连续子串解读。

⌨️ 逐步实现

Step 3 · KMP 找所有出现位置

def find_all_occurrences(s, p):
    n, m = len(s), len(p)
    if m == 0:
        return []
    # 构建 lps
    lps = [0] * m
    k = 0
    for i in range(1, m):
        while k > 0 and p[k] != p[i]:
            k = lps[k-1]
        if p[k] == p[i]:
            k += 1
        lps[i] = k
    # 主匹配
    res = []
    j = 0
    for i in range(n):
        while j > 0 and p[j] != s[i]:
            j = lps[j-1]
        if p[j] == s[i]:
            j += 1
        if j == m:
            res.append(i - m + 1)              # 记录起点
            j = lps[j-1]
    return res

Step 2 · Q1 计数主函数

def count_substr_with_prefix(s, target):
    if not target:
        return sum(range(1, len(s) + 1))
    n, m = len(s), len(target)
    if n < m:
        return 0
    occ = find_all_occurrences(s, target)
    return sum(n - i - m + 1 for i in occ)

Step 4 · Q2 (连续子串解读)

def shortest_containing(s, target):
    if not target:
        return 0
    occ = find_all_occurrences(s, target)
    return len(target) if occ else -1

Q1 解法：每次出现 target 贡献 `(n - end + 1)` 个

子串以 target 为前缀 ⇔ 子串起点是 target 在 s 中的出现位置之一，且子串长度 ≥ |target|。 若 target 在位置 i 出现（i 是 target 的起点），则以这个 target 开头的子串有 n - i - |target| + 1 + |target| - 1 + 1 = n - i 个？

实际上更简单：以位置 i 为起点、且包含 target（target 的起点是 i）的子串，长度可以是 |target|, |target|+1, ..., n-i，共 n - i - |target| + 1 个。

def count_substr_with_prefix(s, target):
    if not target:
        return sum(range(1, len(s) + 1))  # 所有非空子串
    n, m = len(s), len(target)
    if n < m:
        return 0
    occ = find_all_occurrences(s, target)  # KMP / Z
    return sum(n - i - m + 1 for i in occ)


def find_all_occurrences(s, p):
    # KMP
    n, m = len(s), len(p)
    if m == 0:
        return []
    lps = [0] * m
    k = 0
    for i in range(1, m):
        while k > 0 and p[k] != p[i]:
            k = lps[k-1]
        if p[k] == p[i]:
            k += 1
        lps[i] = k
    res = []
    j = 0
    for i in range(n):
        while j > 0 and p[j] != s[i]:
            j = lps[j-1]
        if p[j] == s[i]:
            j += 1
        if j == m:
            res.append(i - m + 1)
            j = lps[j-1]
    return res

Q2 解法：最早出现位置 → 最短子串就是 target 本身（如果存在）

如果 target 在 s 中作为 substring 出现，最短包含 target 的子串就是 target 自己 — 长度 |target|。

def shortest_containing(s, target):
    if not target:
        return 0
    occ = find_all_occurrences(s, target)
    return len(target) if occ else -1

⚠️ 如果 Q2 的"包含 target"是指"包含 target 的所有字符"（multiset 意义，类似 LC 76 Minimum Window Substring），那是另一个题。

澄清这个歧义！

"最小包含 target 的 substring 长度" 有两种解读： 1. substring 意义：target 作为连续子串出现 → 答案就是 |target|（trivial）。 2. multiset 意义：包含 target 的所有字符（顺序不要紧）→ LC 76 滑窗。

这一步必须先问面试官，不然写错算法。

讲解策略

  1. Q1 的关键洞察：算出现位置 → 算贡献。
  2. KMP / Z-algorithm 引出：从朴素 O(nm) 到 O(n+m)。
  3. Q2 澄清歧义：直接问"是连续子串还是字符集合"，体现对题目敏感度。
  4. 如果是 multiset 意义：滑动窗口 O(n)，参考 LC 76 解法。
  5. 复杂度：Q1 O(n+m)，Q2 同。

常见坑

  • 没澄清 Q2 含义：直接写错算法。
  • KMP 失败函数写错：常见 bug 来源；不熟就用 Python 的 s.find 配 while 找所有位置，朴素 O(nm) 但能 work。
  • target 空串：是 corner case，先确认。
  • target = s 完整字符串：贡献是 1（不是 0），别写错边界。
