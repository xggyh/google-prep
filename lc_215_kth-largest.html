LC 215. Kth Largest Element in an Array 数组中的第 K 个最大元素

类别：📚 堆 / 扫描线 · 难度：Med · 标记：⭐ Google 直接对应

📚 堆 / 扫描线 Med ⭐ Google 直接对应

题目概览

🏷️ 类别：堆 / Quickselect | 难度：Medium | 标记：⭐

题目

EN: Given an unsorted array nums and integer k, return the kth largest element. Solve without sorting.

中文：无序数组中找第 k 大元素，不能用排序。

澄清问题

  • EN: 1-indexed k? / Duplicates allowed? / Modify input ok? / Memory constraints?
  • 中：k 是 1-indexed 吗？有重复吗？可以原地修改吗？有内存限制吗？

🧠 整体思路

两条路： 1. Min-heap size k：O(n log k) 时间，O(k) 空间 — 流式数据友好。 2. Quickselect 随机 pivot：O(n) 期望时间，O(log n) 栈 — 一次性数据更快。

Quickselect 关键：选 pivot → 三路划分 → 看 target 落哪段递归。

🪜 思路拆解（Quickselect 版）

Step 1 · 索引转换 - 第 k 大 = 第 (n-k) 小（0-indexed target）。

Step 2 · 随机 pivot - random.randint(lo, hi) 选 pivot。 - 已排序输入 → 第一个或最后一个 pivot 会退化。

Step 3 · 三路划分（Dutch flag） - 三段：< pivot | == pivot | > pivot。 - 处理重复元素更高效。

Step 4 · 根据 target 递归 - target < lt → 左半。 - target > gt → 右半。 - 否则 → 落在 == 段，返回 pivot。

⌨️ 逐步实现

Step 1 · 入口

import random

def findKthLargest(nums, k):
    nums = nums[:]
    target = len(nums) - k

Step 2-4 · Quickselect

    def select(lo, hi):
        if lo == hi:
            return nums[lo]
        pivot = nums[random.randint(lo, hi)]
        lt, i, gt = lo, lo, hi
        while i <= gt:
            if nums[i] < pivot:
                nums[lt], nums[i] = nums[i], nums[lt]
                lt += 1; i += 1
            elif nums[i] > pivot:
                nums[gt], nums[i] = nums[i], nums[gt]
                gt -= 1
            else:
                i += 1
        if target < lt:
            return select(lo, lt - 1)
        elif target > gt:
            return select(gt + 1, hi)
        return pivot

    return select(0, len(nums) - 1)

Min-heap 版（备用，更易理解）

import heapq

def findKthLargest_heap(nums, k):
    heap = []
    for x in nums:
        heapq.heappush(heap, x)
        if len(heap) > k:
            heapq.heappop(heap)
    return heap[0]                          # 堆顶就是第 k 大

讲解策略

  1. 列三种方法：sort / heap / quickselect — 主动给 trade-off。
  2. 选 quickselect 期望最优。
  3. 三路 partition 处理重复元素的好处。
  4. 提 median-of-medians 最坏 O(n) 但常数大。

常见坑

  • 固定 pivot：已排序输入 O(n²)。
  • Lomuto + 全相同数组：O(n²)。三路更稳。
  • 递归终止 lo > hi：边界判断要 lo == hi 返回。
