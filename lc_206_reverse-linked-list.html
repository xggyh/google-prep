LC 206. Reverse Linked List 反转链表（含 hash 验证变体）

类别：🎮 模拟 / 链表 / 并发 · 难度：Easy · 标记：➕ 额外推荐

🎮 模拟 / 链表 / 并发 Easy ➕ 额外推荐

题目概览

🏷️ 类别：模拟 / 链表 | 难度：Easy | 标记：➕

题目

EN: Reverse a singly linked list. Variant from Google: each node has a hash field where hash = val + hash(next). After reverse, verify the hash chain still holds (hash now propagates in the new direction).

中文：反转单链表。Google 变体：每个节点有 hash = val + hash(next)。反转后 hash 也要重新计算成新方向的链。

澄清问题

  • EN: In-place? / Iterative or recursive ok? / What's hash of null? (Usually 0)
  • 中：原地反转？迭代还是递归？空节点的 hash 是什么？

🧠 整体思路

反转链表 = 经典三指针迭代：prev / cur / nxt。

Hash 验证：原始链表 hash[i] = val[i] + hash[i+1]，反转后 hash[i] = val[i] + hash[i-1]（在新链表里）。所以反转过程中重新计算每个节点的 hash。

关键 idiom：边反转边重算 hash，无需再扫一遍。

🪜 思路拆解

Step 1 · 边界 - head 为 None → 直接返回。

Step 2 · 三指针 + hash 累加 - 维护 prev = None，cur = head，prev_hash = 0。 - 每一步： - 记下 nxt = cur.next。 - cur.next = prev（反转指针）。 - cur.hash = cur.val + prev_hash（重算 hash）。 - 更新 prev = cur，prev_hash = cur.hash，cur = nxt。

Step 3 · 返回新 head - 循环结束时 prev 就是新 head。

⌨️ 逐步实现

Step 1 · 边界

class Node:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next
        self.hash = 0

def reverseWithHash(head):
    if not head:
        return None

Step 2 · 三指针 + hash 重算

    prev = None
    cur = head
    prev_hash = 0
    while cur:
        nxt = cur.next                     # 保存下一个
        cur.next = prev                    # 反转指针
        cur.hash = cur.val + prev_hash     # 重新计算 hash
        prev_hash = cur.hash               # 推进 prev_hash
        prev = cur
        cur = nxt

Step 3 · 返回

    return prev

Python 完整实现

def reverseWithHash(head):
    if not head:
        return None
    prev, cur, prev_hash = None, head, 0
    while cur:
        nxt = cur.next
        cur.next = prev
        cur.hash = cur.val + prev_hash
        prev_hash = cur.hash
        prev = cur
        cur = nxt
    return prev

讲解策略

  1. 反转 + 重算 hash 一次走完，不要两遍扫。
  2. prev_hash 是关键状态（每步累加），不要漏。
  3. 复杂度 O(n) time，O(1) space。

常见坑

  • hash 用旧 hash：反转后 next 已经变方向了，必须用 prev_hash 重算。
  • prev_hash 初始 0：约定"空 / 末尾的 hash 是 0"。
  • 三指针顺序错：必须先存 nxt 再改 next，否则丢失链。
