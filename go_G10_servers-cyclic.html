Google 原创 G10. Servers for Cyclic Tasks 循环任务服务器分配

类别：📚 堆 / 扫描线 · 难度：Hard · 标记：🆕 Google 原创

📚 堆 / 扫描线 Hard 🆕 Google 原创

题目概览

🏷️ 类别：堆 / 扫描线 | 难度：Hard | 标记：Google 原创 | 来源：L3/L4 编译 #21

题目

EN: Repeating/cyclic tasks (start, duration) — each task runs forever, restarting every period. Compute min number of servers.

中文：周期性任务 (start, duration)，按某 period 永远循环。求所需最少服务器数。

🧠 整体思路

关键洞察 — 找一个 period 内的峰值：因为周期性，最高同时占用数 = 一个周期内的最高 overlap。

做法： - 把每个任务的所有"start..start+duration" 区间放到 period 周期内（mod 处理）。 - 用扫描线找峰值。

注意：如果 duration > period 会有 wrap-around。

🪜 思路拆解

Step 1 · 周期 P 已知，每个 task 映射到 [start mod P, (start+duration) mod P] Step 2 · 扫描线 +1/-1 Step 3 · 处理 wrap-around：跨周期的拆成两段

⌨️ 逐步实现

def min_servers_cyclic(tasks, period):
    events = []
    for start, duration in tasks:
        s = start % period
        e = (start + duration) % period
        if s < e:
            events.append((s, 1))
            events.append((e, -1))
        else:
            # wrap around
            events.append((s, 1))
            events.append((period, -1))
            events.append((0, 1))
            events.append((e, -1))
    events.sort(key=lambda x: (x[0], x[1]))
    cur = max_cur = 0
    for _, delta in events:
        cur += delta
        max_cur = max(max_cur, cur)
    return max_cur

讲解策略

  1. 周期性 → 看一个周期。
  2. wrap-around 拆两段。
  3. 扫描线模板。

常见坑

  • duration > period 时所有时刻都在跑（需要 ceiling）。
  • 端点是 < 还是 <=：影响 tie-break。
