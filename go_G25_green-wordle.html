Google 原创 G25. Green-Only Wordle 仅绿色反馈的 Wordle

类别：🎮 模拟 / 链表 / 并发 · 难度：Med · 标记：🆕 Google 原创

🎮 模拟 / 链表 / 并发 Med 🆕 Google 原创

题目概览

🏷️ 类别：模拟 / 链表 / 并发 / 启发式 | 难度：Medium | 标记：Google 原创 | 来源：L3/L4 编译 #9

题目

EN: Choose optimal guesses from a dictionary to find hidden 5-letter word using only exact-match (green) feedback. Each guess returns "for each position, is it correct".

中文：从字典选 guess，反馈只告诉每个位置对不对（绿色）。求最快猜出隐藏单词。

🧠 整体思路

信息论启发式：每次选能最大化区分剩余候选的单词。

具体：对每个候选 guess g，计算"如果 g 是答案的话，反馈模式是什么"，把候选按反馈分桶。最大桶最小的 g 是最好的下一猜。

类似经典 Wordle solver 的 minimax 策略。

🪜 思路拆解

Step 1 · 维护 candidates 集合（初始 = 整个字典） Step 2 · 选 best guess (minimize worst-case partition) Step 3 · 收到反馈后过滤 candidates Step 4 · 重复直到 candidates == 1 (得到答案)

def green_only_wordle(words):
    candidates = set(words)
    guesses = []

    def feedback(guess, target):
        return tuple(g == t for g, t in zip(guess, target))

        # Minimax: minimize max partition size
        best = None
        best_max = float('inf')
        for g in words:                    # 允许从全字典选 (信息论 guess)
            buckets = {}
            for c in candidates:
                fb = feedback(g, c)
                buckets[fb] = buckets.get(fb, 0) + 1
            worst = max(buckets.values())
            if worst < best_max:
                best_max, best = worst, g
        return best

讲解策略

  1. 信息论 minimax：每步最大化区分度。
  2. 时间复杂度 O(|words|² · k) per guess，离线预处理可加速。
  3. 简化：随机 guess 也能在 O(log |D|) 期望步内找到。

常见坑

  • 真实 Wordle 还有黄色（错位置）反馈，这题只有绿。
  • 字典可能没答案 → 候选空。
