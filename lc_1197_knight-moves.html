LC 1197. Minimum Knight Moves 进击的骑士

类别：🗺️ 图 / BFS / DFS · 难度：Med · 标记：➕ 额外推荐

🗺️ 图 / BFS / DFS Med ➕ 额外推荐

题目概览

🏷️ 类别：图 / BFS | 难度：Medium | 标记：➕

题目

EN: An infinite chessboard with knight at (0, 0). Return minimum knight moves to reach (x, y). Knight moves in an "L" shape (8 directions).

中文：无限大棋盘，骑士在 (0,0)，每步走"日"字（8 个方向）。求到 (x, y) 的最少步数。

澄清问题

  • EN: Range of x, y? (-300 ≤ x, y ≤ 300 typically) / Always reachable?
  • 中：x, y 的范围？是否一定可达？

🧠 整体思路

BFS 求最少步数。但棋盘无限大 → 不能用二维数组当 visited，用 set。

关键优化 — 对称性剪枝： - 骑士问题是 4 象限对称的：x = abs(x); y = abs(y) 后只在第一象限搜。 - 进一步：搜索区间可以收紧到 x, y >= -2 附近（避免无意义的"绕远"）。

也可以用双向 BFS / A*，但单向 BFS 加对称剪枝够用。

🪜 思路拆解

Step 1 · 对称化目标 - x, y = abs(x), abs(y)。

Step 2 · BFS 初始化 - 队列从 (0, 0) 起，步数 0。 - visited = {(0, 0)}。

Step 3 · 8 方向移动 - 骑士的 8 个偏移：[(2,1),(1,2),(-1,2),(-2,1),(-2,-1),(-1,-2),(1,-2),(2,-1)]。

Step 4 · 剪枝 - 跳过坐标过远（e.g., nx < -2 or ny < -2）— 避免无意义远跳。

Step 5 · 命中目标返回步数 - 每层扩展时检查是否到达 (x, y)。

⌨️ 逐步实现

Step 1-3 · 初始化 + 8 方向

from collections import deque

def minKnightMoves(x, y):
    x, y = abs(x), abs(y)                     # Step 1: 对称化
    moves = [(2,1),(1,2),(-1,2),(-2,1),
             (-2,-1),(-1,-2),(1,-2),(2,-1)]
    queue = deque([(0, 0, 0)])                # (cur_x, cur_y, steps)
    visited = {(0, 0)}

Step 4-5 · BFS 主循环 + 剪枝

    while queue:
        cx, cy, steps = queue.popleft()
        if (cx, cy) == (x, y):
            return steps
        for dx, dy in moves:
            nx, ny = cx + dx, cy + dy
            # Step 4 剪枝: 不要走太远
            if (nx, ny) not in visited and nx >= -2 and ny >= -2:
                visited.add((nx, ny))
                queue.append((nx, ny, steps + 1))
    return -1

Python 完整实现

from collections import deque

def minKnightMoves(x, y):
    x, y = abs(x), abs(y)
    moves = [(2,1),(1,2),(-1,2),(-2,1),(-2,-1),(-1,-2),(1,-2),(2,-1)]
    queue = deque([(0, 0, 0)])
    visited = {(0, 0)}
    while queue:
        cx, cy, steps = queue.popleft()
        if (cx, cy) == (x, y):
            return steps
        for dx, dy in moves:
            nx, ny = cx + dx, cy + dy
            if (nx, ny) not in visited and nx >= -2 and ny >= -2:
                visited.add((nx, ny))
                queue.append((nx, ny, steps + 1))
    return -1

讲解策略

  1. 关键洞察：BFS 求最少步数。
  2. 对称化：abs 让搜索空间缩 4 倍。
  3. 剪枝防止无限扩展：限制 >= -2。
  4. 进阶可提双向 BFS，进一步降复杂度。

常见坑

  • 不剪枝：visited 集合无限增长，TLE 或 MLE。
  • 对称化后忘记目标也转换：x = abs(x); y = abs(y) 必须做。
  • -2 边界：太严会漏解（骑士可能短暂跳负坐标）；太松会爆内存。
