14. 最长匹配分词（LC 139/140 类）

来源：Google r1 · LC: LC 139/140 类 🟡 变体 · 难度: Med

Trie Greedy String 🟡 变体 Med

题目概览

来源：Google r1 原描述：一段 text + 一个 dictionary (key:id)。按 longest match 贪心切词，match 上输出 id，没 match 的字符原样吐出。Follow-up 改了切词优先级。

题目

EN: Given a text and a dict: {token -> id}. Tokenize text greedily by longest match: at each position, find the longest token in dict that matches starting from this position; if found, emit its id; otherwise emit the single character literally. Advance and repeat.

中文：给一段文本和字典 (token → id)。从左到右扫描，每个位置贪心取最长能匹配的字典 token，输出 id；如果没有 match，原样输出当前字符；继续。

澄清问题

  • EN: Case-sensitive? / Are dict keys all non-empty? / Can dict tokens overlap (same prefix)? / Output format — list, or string with markers? / Follow-up — what's the new priority rule? / Dict size and text length?
  • 中：大小写敏感？字典 key 是否非空？是否要 case-fold？输出格式（list / 字符串 / 带标记）？Follow-up 的新优先级规则是什么？字典大小和文本长度的量级？

Python 实现（HashMap 法 — 简单）

def tokenize(text, vocab):
    """vocab: dict[str, int]"""
    max_len = max((len(k) for k in vocab), default=0)
    n = len(text)
    out = []
    i = 0
    while i < n:
        matched = False
        for L in range(min(max_len, n - i), 0, -1):
            tok = text[i:i+L]
            if tok in vocab:
                out.append(vocab[tok])
                i += L
                matched = True
                break
        if not matched:
            out.append(text[i])
            i += 1
    return out

Trie 版本（推荐讲）

class TrieNode:
    def __init__(self):
        self.children = {}
        self.token_id = None  # 是否是 token 结尾，及其 id


def build_trie(vocab):
    root = TrieNode()
    for tok, tid in vocab.items():
        node = root
        for ch in tok:
            node = node.children.setdefault(ch, TrieNode())
        node.token_id = tid
    return root


def tokenize_trie(text, vocab):
    root = build_trie(vocab)
    n = len(text)
    out = []
    i = 0
    while i < n:
        node = root
        j = i
        last_match = None  # (end_pos, token_id)
        while j < n and text[j] in node.children:
            node = node.children[text[j]]
            j += 1
            if node.token_id is not None:
                last_match = (j, node.token_id)
        if last_match:
            out.append(last_match[1])
            i = last_match[0]
        else:
            out.append(text[i])
            i += 1
    return out

讲解策略

  1. 先写朴素版：HashMap + 倒序枚举长度。复杂度 O(N · max_token_len)。
  2. Trie 优化：解释为什么 trie 能把"试遍每个长度"省成"一次走"，复杂度 O(N · avg_depth)。
  3. Follow-up（切词优先级改变）：常见变体 — (a) shortest match 优先；(b) 频率高的优先；(c) BPE 风格按 merge 顺序。能立刻调整算法是加分项。
  4. 类比真实 NLP tokenizer：BPE / WordPiece — 让面试官知道你懂这个领域。
  5. dry run：用 text="abcdef", dict={"ab":1, "abc":2, "de":3} 走一遍。

常见坑

  • 没问"贪心"还是"全局最优"：贪心 longest match 不一定全局最优分词数，面试官可能问 DP 版（LC 140）。
  • 重叠字典 key：dict={"a":1, "ab":2}，position 0 该选哪个？题目说 longest 就选 "ab"。
  • Trie 构建 O(总 token 长度)：要算进总复杂度。
  • Follow-up 没准备：跨各种切词规则都要能讲清适应改法。
