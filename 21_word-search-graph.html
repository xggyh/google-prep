21. Word Search 图变体（LC 79 变体）

来源：果家 onsite Coding 1 · LC: LC 79 变体 🟡 变体 · 难度: Med

DFS Backtrack Graph 🟡 变体 Med

题目概览

来源：果家 onsite Coding 1 原描述："lc 起舅 (79) 变体，graph is a list of nodes."

题目

EN: Standard LC 79 is on a 2D grid. Variant: the "board" is now a general graph (list of nodes, each with a character and edges to neighbors). Given a word, return whether the word can be spelled by a path in the graph (no node reused).

中文：LC 79 的网格换成了一般图（节点列表 + 邻接关系）。判断一个 word 能不能在图上拼出来（一笔走出来，不重复用节点）。

澄清问题

  • EN: Graph directed or undirected? / Can two nodes have the same character? / Word length range? / Number of nodes? / Can the path revisit edges? / What if word is empty?
  • 中：图是有向还是无向？两个节点能有相同字符吗？word 长度？节点数？路径可以重复用边吗（一般不行）？word 空怎么处理？

🧠 整体思路

LC 79 在 2D grid 上的算法是 DFS + 4 邻接 + visited backtracking。这里把 grid 换成一般 graph，算法骨架完全一样，只是邻接关系从"4 个方向"变成"邻接表"。

搜索路径： - 选一个起始节点（字符 = word[0]）。 - DFS：当前节点字符匹配 word[idx]？匹配就标记 visited，递归到 word[idx+1] 试所有未访问邻居。 - 注意 backtrack：递归返回时把 visited 标记移除。

枚举所有起点：因为图里多个节点可能字符 = word[0]，每个都要试。

复杂度：每个节点作为起点 → O(V · branching^|word|)。比 grid 版分支更大（grid 是 4，graph 可能更高）。

🪜 思路拆解

Step 1 · 数据结构（先和面试官敲定签名） - nodes: dict[node_id] -> char：每个节点的字符。 - edges: dict[node_id] -> list[node_id]：邻接表。 - 是有向还是无向？要问。

Step 2 · 边界 - 空 word：return True（"空串总能拼出来"约定，但要确认）。 - 空 graph：return False。

Step 3 · DFS 函数签名 - dfs(node, idx)：从 node 出发，匹配 word[idx]。 - 返回 True 表示能完成 word[idx:] 的匹配。

Step 4 · DFS 主体 - 字符不匹配 → 立刻 False。 - 到 word 末尾 (idx == len(word) - 1) → True。 - 否则标记 visited，遍历邻居： - 跳过已访问。 - 递归调用，任一返回 True 就 True。 - 回溯：无论成功失败都 remove 当前节点 from visited（成功也要 remove，不然 visited 会污染下次起点）。

Step 5 · 枚举所有起点 - for node in nodes：字符 = word[0] 的都试一次。 - 第一个返回 True 就 True。

Step 6 · Backtrack 的对称性 - 进 DFS 时 add，出 DFS 时 remove。 - 三个分支（base case 返回、找到答案返回、遍历完返回）都要 remove。

⌨️ 逐步实现

Step 1-2 · 框架

def exist_in_graph(nodes, edges, word):
    if not word:
        return True
    visited = set()

Step 3-4 · DFS + Backtrack

    def dfs(node, idx):
        if nodes[node] != word[idx]:
            return False
        if idx == len(word) - 1:
            return True

        visited.add(node)
        for nb in edges.get(node, []):
            if nb in visited:
                continue
            if dfs(nb, idx + 1):
                visited.remove(node)            # 成功也要回溯
                return True
        visited.remove(node)                    # 失败回溯
        return False

Step 5 · 枚举起点

    return any(nodes[n] == word[0] and dfs(n, 0) for n in nodes)

Python 完整实现

def exist_in_graph(nodes, edges, word):
    """nodes: dict[node_id] -> char
       edges: dict[node_id] -> list[node_id]"""
    if not word:
        return True
    target = word[0]
    visited = set()

    def dfs(node, idx):
        if nodes[node] != word[idx]:
            return False
        if idx == len(word) - 1:
            return True
        visited.add(node)
        for nb in edges[node]:
            if nb not in visited and dfs(nb, idx + 1):
                visited.remove(node)
                return True
        visited.remove(node)
        return False

    return any(nodes[n] == target and dfs(n, 0) for n in nodes)

讲解策略

  1. 指出与 LC 79 的同构性：网格其实就是特殊的"4-邻接图"。
  2. DFS + backtrack：visited 进出要对称。
  3. 优化：把字符相同的节点 group 起来，搜索时只考虑 word 中下一个字符对应的候选 → 减少分支。
  4. 复杂度：最坏 O(|nodes| · 4^|word|)（一般图分支可能更高）。
  5. 果家面经原话："函数签名可以自己定义，面试官会一直变需求"，要主动 propose 数据结构（nodes dict, edges dict）。

常见坑

  • visited 没回退：进 DFS 加，回来时一定要删。
  • 节点数 = 字符数？还是不同节点同字符也可以？ 必须问。
  • 图可能有环：visited 防止死循环。
  • 入口枚举：不只从一个起点，所有字符 = word[0] 的节点都要试。
