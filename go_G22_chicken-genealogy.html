Google 原创 G22. Chicken Genealogy 鸡的家族关系（LCA / 祖先判定）

类别：🌳 树 / BST · 难度：Med · 标记：🆕 Google 原创

🌳 树 / BST Med 🆕 Google 原创

题目概览

🏷️ 类别：树 / BST / LCA | 难度：Medium | 标记：Google 原创 | 来源：L3/L4 编译 #15

题目

EN: Given parent-child pairs, determine if two specific nodes are related (one is ancestor of the other OR share common ancestor).

中文：给定父子关系，判断两个节点是否有血缘（一个是另一个的祖先，或共同祖先存在）。

🧠 整体思路

两种判定： 1. 同一棵树（forest 中同一连通分量） → Union-Find。 2. LCA 存在 → 沿 parent 链向上找。

如果题目是"在同一棵树里"，UF 最直接。

🪜 思路拆解（UF 版）

Step 1 · 建 UF Step 2 · 对每对 (a, b)，find(a) == find(b)?

⌨️ 逐步实现

class UF:
    def __init__(self):
        self.parent = {}
    def find(self, x):
        if x not in self.parent:
            self.parent[x] = x
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x
    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx != ry:
            self.parent[rx] = ry

def is_related(parent_child_pairs, a, b):
    uf = UF()
    for p, c in parent_child_pairs:
        uf.union(p, c)
    return uf.find(a) == uf.find(b)

讲解策略

  1. "有血缘" = "同一棵树"。
  2. UF 是这类判定的最简方案。
  3. 如果需要找 LCA 本身，建 parent map 沿链上升。

常见坑

  • 节点可能没在任何 pair 中出现 → 单独算一组。
  • "祖先关系" vs "亲戚关系" 严格定义先问。
