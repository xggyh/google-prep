LC 140. Word Break II 单词拆分 II

类别：🔤 字符串 / 解析 / 栈 · 难度：Hard · 标记：🔄 LC 变体

🔤 字符串 / 解析 / 栈 Hard 🔄 LC 变体

题目概览

🏷️ 类别：字符串 / Backtrack + Memoization | 难度：Hard | 标记：🔄

题目

EN: Same as LC 139, but return all possible sentences.

中文：和 LC 139 一样，但返回所有可能的拆分方式。

澄清问题

  • 同 139。
  • EN: Are duplicate sentences possible / desired? / Max output size?
  • 中：可能有重复句子吗？输出数量上界？

🧠 整体思路

Backtrack + Memoization：dfs(i) 返回 s[i:] 的所有拆分方式（list of strings）。

关键：用 memo 避免重复计算（同一个起点会被多次访问）。

复杂度：指数级（最坏 2^n 个结果），但 memo 让中间结果共享。

🪜 思路拆解

Step 1 · 字典转 set - O(1) 查询。

Step 2 · 手写 memo 字典 - memo: dict[int, list[str]]，key 是起点 start，value 是该起点的所有拆分结果。 - 为什么手写而不用 @lru_cache： 1. 面试官常希望看到你主动管理 memo，证明你理解记忆化。 2. lru_cache 是 Python 特定语法，C++/Java 没有等价物，写手动版可移植。 3. 手写更容易讲清楚"为什么这步能 memoize"。

Step 3 · 递归 dfs(start) → 所有拆分 - 入口先查 memo，命中直接返回。 - 终止：start == n 返回 [""]（空尾巴 — 这是关键 idiom，让上层能 join）。 - 遍历 end 从 start+1 到 n： - word = s[start:end] 在字典？ - 递归 tails = dfs(end)。 - 把 word + 每个 tail 拼起来加到 result。 - 递归返回前写入 memo。

Step 4 · 拼接细节 - tail 为空 → 直接用 word（避免末尾多余空格）。 - tail 非空 → word + " " + tail。

⌨️ 逐步实现

Step 1-2 · 初始化（手写 memo 替代 lru_cache）

def wordBreak(s, wordDict):
    word_set = set(wordDict)
    n = len(s)
    memo = {}                                # 手动 memo: start -> list[str]

Step 3 · 递归 + 手动 memo

    def dfs(start):
        if start in memo:                    # memo 命中直接返回
            return memo[start]
        if start == n:
            return [""]                      # 空尾巴, 让上层 join 不出错

        result = []
        for end in range(start + 1, n + 1):
            word = s[start:end]
            if word in word_set:
                for tail in dfs(end):
                    if tail:
                        result.append(word + " " + tail)
                    else:
                        result.append(word)  # tail 空, 不加空格
        memo[start] = result                 # 写入 memo 后返回
        return result

Step 4 · 调用入口

    return dfs(0)

Python 完整实现

def wordBreak(s, wordDict):
    word_set = set(wordDict)
    n = len(s)
    memo = {}

    def dfs(start):
        if start in memo:
            return memo[start]
        if start == n:
            return [""]
        result = []
        for end in range(start + 1, n + 1):
            word = s[start:end]
            if word in word_set:
                for tail in dfs(end):
                    if tail:
                        result.append(word + " " + tail)
                    else:
                        result.append(word)
        memo[start] = result
        return result

    return dfs(0)

讲解策略

  1. 比 139 复杂：要枚举所有路径而非只判存在。
  2. 手写 memo 字典：明确告诉面试官 "I'll manage the memo manually so you can see the logic"。
  3. Memo 的 key 是 start，value 是 s[start:] 的所有拆分 — 子问题独立于上下文。
  4. 输出最坏指数级（2^n 种拆分），要主动说。
  5. 提优化：可以先跑一遍 LC 139 判定能否拆，能拆再跑 dfs，避免无用搜索（在反例输入如 s="aaaaaaaaab", dict=["a","aa","aaa",...] 中救命）。

常见坑

  • 忘记写 memo：递归会指数爆炸。
  • 空尾巴处理：dfs(n) 返回 [""] 而不是 []，否则上层 for tail in dfs(end) 直接跳出，结果空。
  • 拼接空格：tail 为空时不加空格，不然末尾多 1 个空格。
  • memo 用 list 当 key：start 是 int，没问题；但如果改用 substring 当 key（如 memo[s[start:]]），内存爆炸。
