🔑 从面经里反推的 Strong Hire 要点

  • 澄清问题：多份成功面经把这一点列为加分项（"clarifications helped" / "communication was the key"）。
  • Dry run 至少做一遍：被多份 Hire 评语点名。
  • "Minor syntax issue" 真的能挂人（NG 那位的拒信原话）— 写慢一点，写干净一点。
  • Brute force → Optimized 的递进非常加分（SWE II L3 那题特别提到）。
  • Time/Space complexity 主动给，每题都要。
  • Google Doc 真的没补全，平时关掉 IDE 智能提示练。

🧭 通用应试流程（每题都用得上）
时间段动作避免
0:00–0:03复述题目 + 举一个具体例子跳过直接开始问
0:03–0:08问 3–5 个澄清问题，确认 scope连珠炮 10 个问题
0:08–0:13给 brute force + 优化方向（先要面试官点头）直接写最优解
0:13–0:18讨论数据结构选型 + 复杂度不讨论 trade-off
0:18–0:35边写边讲，每写完一段先 dry run 一遍沉默写代码
0:35–0:40主动选 1-2 个 test case 完整 dry run等面试官说"跑一下"
0:40–0:45复杂度分析 + Follow-up 讨论 + 反问不分析复杂度
