20. Detonate the Maximum Bombs（LC 2101）

来源：Onsite NG R2 · LC: LC 2101 ✅ 高置信 · 难度: Med

BFS Graph Geometry ✅ 高置信 Med

题目概览

来源：Google onsite NG 被拒 R2 原描述：LC 2101 detonate bomb 变数题。"刷过很多次秒杀。"

题目

EN: You're given bombs[i] = [x, y, r]. Detonating bomb i ignites all bombs within distance r from (x, y), recursively. Return the max number of bombs you can detonate by choosing one starting bomb.

中文：每个炸弹 [x, y, r]。引爆某个炸弹会引爆所有在它半径 r 内的炸弹（递归）。选一个起点，求最多能引爆几个。

澄清问题

  • EN: Are coordinates and radius integers? / Can two bombs be at same position? / Is the chain reaction strictly "within distance r" — at distance exactly r counted? / Bomb count?
  • 中：坐标和半径是整数还是浮点？两个炸弹可以同位置吗？距离正好 = r 算引爆吗？n 的范围（决定算法）？

🧠 整体思路

这题有 2 个坑，识别出来就是 strong hire： 1. 不是无向图：i 能引爆 j（dist ≤ r_i）不蕴含 j 能引爆 i（需要 dist ≤ r_j）— 半径不对称！ 2. 距离比较用平方：避免 sqrt 浮点误差。

算法 = 建有向图 + 枚举起点 BFS： - 建图：对所有 (i, j) 检查"i 能引爆 j 吗"，能就加边 i → j。 - 枚举每个炸弹 i 作为起点跑 BFS，记录可达炸弹数。 - 返回所有起点的最大值。

复杂度：建图 O(n²)，n 个 BFS 各 O(n²) → O(n³)。n ≤ 100 时完全 OK。

🪜 思路拆解

Step 1 · 建有向图 - 遍历所有 (i, j)，i ≠ j。 - 计算 dist² = (x1-x2)² + (y1-y2)²。 - 比较 dist² <= r1²（r1 是 i 的半径）。 - 是就加边 i → j。

Step 2 · 从单个起点 BFS - 标准 BFS，用 set 记录已 visited。 - 返回 len(visited)。

Step 3 · 枚举所有起点 + 取最大 - return max(bfs(i) for i in range(n))。

Step 4 · 关键 idiom：距离平方 - 不要写 math.sqrt(...) <= r。 - 写 dx*dx + dy*dy <= r*r。 - 避免浮点 / 提速。

Step 5 · Edge cases - 两个炸弹同位置：必须问。如果同坐标，互相一定能引爆（dist=0 ≤ 任何 r）。 - 单个炸弹：return 1。 - 半径 0：只能炸自己。

⌨️ 逐步实现

Step 1 · 建有向图

from collections import defaultdict, deque

def maximumDetonation(bombs):
    n = len(bombs)
    graph = defaultdict(list)
    for i in range(n):
        x1, y1, r1 = bombs[i]
        for j in range(n):
            if i == j:
                continue
            x2, y2, _ = bombs[j]
            # i 能引爆 j ?
            if (x1 - x2) ** 2 + (y1 - y2) ** 2 <= r1 * r1:
                graph[i].append(j)

Step 2 · 单起点 BFS

    def bfs(start):
        visited = {start}
        queue = deque([start])
        while queue:
            u = queue.popleft()
            for v in graph[u]:
                if v not in visited:
                    visited.add(v)
                    queue.append(v)
        return len(visited)

Step 3 · 枚举起点

    return max(bfs(i) for i in range(n))

Python 完整实现（建图 + BFS）

from collections import defaultdict, deque


def maximumDetonation(bombs):
    n = len(bombs)
    graph = defaultdict(list)
    for i in range(n):
        x1, y1, r1 = bombs[i]
        for j in range(n):
            if i == j:
                continue
            x2, y2, _ = bombs[j]
            # bomb i 能引爆 bomb j 吗
            if (x1 - x2) ** 2 + (y1 - y2) ** 2 <= r1 * r1:
                graph[i].append(j)

    def bfs(start):
        visited = {start}
        queue = deque([start])
        while queue:
            u = queue.popleft()
            for v in graph[u]:
                if v not in visited:
                    visited.add(v)
                    queue.append(v)
        return len(visited)

    return max(bfs(i) for i in range(n))

讲解策略

  1. 关键 insight："i 引爆 j" 不对称（i 的半径决定它能引爆谁），所以是有向图。
  2. 建图 O(n²)，每个起点 BFS O(n²)，总 O(n³)。n ≤ 100 完全 OK。
  3. 距离比较用平方：避免 sqrt 浮点误差。
  4. 想优化到 O(n² log n)：用 KD-tree / spatial index，但 n=100 没必要。
  5. dry run：3 个炸弹的小例子，强调"有向"。
  6. "秒杀"的关键：把"有向图"和"距离平方"两个 trap 明确指出来，面试官就放心。

常见坑

  • 以为是无向图：i.r > dist(i,j) 不蕴含 j.r > dist(i,j)。
  • sqrt 比较：浮点误差导致漏算。
  • < vs <=：恰好 = r 算不算？默认 LC 是 <=，但要问。
  • 从每个起点重新 BFS 看似浪费，其实必须 — 因为不同起点的连通片不同。
