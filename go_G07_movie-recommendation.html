Google 原创 G07. Movie Recommendation Graph 电影推荐图

类别：🗺️ 图 / BFS / DFS · 难度：Easy · 标记：🆕 Google 原创

🗺️ 图 / BFS / DFS Easy 🆕 Google 原创

题目概览

🏷️ 类别：图 / BFS + 排序 | 难度：Easy | 标记：Google 原创 | 来源：L3/L4 编译 #18

题目

EN: Set of movies + undirected similarity graph (edges = similar). Given user's watched movies and k, recommend top k based on similarity.

中文：电影 + 相似图，给定用户已看的电影集，推荐 top k 个相似但未看的。

🧠 整体思路

多源 BFS：从用户已看的电影出发，BFS 扩散。距离越近 → 相似度越高。

收集距离最近的 k 个（排除已看）。

🪜 思路拆解

Step 1 · 多源 BFS：已看电影都入队，距离 0 Step 2 · 扩散记录每个未看电影的最小距离 Step 3 · 按距离排序，取 top k

⌨️ 逐步实现

from collections import deque

def recommend(graph, watched, k):
    seen = set(watched)
    dist = {}
    q = deque((m, 0) for m in watched)
    while q:
        m, d = q.popleft()
        for nb in graph[m]:
            if nb not in seen and nb not in dist:
                dist[nb] = d + 1
                q.append((nb, d + 1))
    return [m for m, _ in sorted(dist.items(), key=lambda x: x[1])][:k]

讲解策略

  1. 多源 BFS 求"最近相似"。
  2. 排除已看 set。
  3. Follow-up：边权 = 相似度 → Dijkstra。

常见坑

  • 用户已看的不要推荐。
  • 同距离打破平局：可加"流行度"二级 key。
