LC 642. Design Search Autocomplete System 设计搜索自动补全系统

类别：🏗️ OOP 设计 · 难度：Hard · 标记：➕ 额外推荐

🏗️ OOP 设计 Hard ➕ 额外推荐

题目概览

🏷️ 类别：OOP 设计 / Trie + Heap | 难度：Hard | 标记：➕

题目

EN: - __init__(sentences, times): history of sentences with their hot degrees. - input(c): user types a char. Return top 3 historical sentences starting with current prefix (sorted by hot degree desc, then ascii asc). If c == '#', save current input as new sentence (or increment if exists), return [].

中文：搜索补全系统。输入字符流，返回当前前缀的热度 top 3。# 结束一次输入并入库。

澄清问题

  • EN: Top-3 means up to 3 (could be < 3)? / How to handle hot degree ties (ASCII order)?
  • 中：top 3 是"最多 3 个"还是"恰好 3 个"？热度相同时按 ASCII 排？

🧠 整体思路

Trie + 每个 node 维护 "这下面所有句子的 (hot, sentence) 集合"： - 用 dict 存 sentence → hot，每次 input 扫一遍这个 dict 取 top 3 即可（k=3 很小，扫描成本可接受）。 - 更高级实现：每个 trie node 缓存 sorted top-3，O(1) 查询，但实现复杂。

简化版（推荐面试用）： - 维护 trie + sentence_to_hot dict。 - 当前 prefix 走到 trie node。 - 那个 node 下的所有句子 → 排序取 top 3。

为了性能，可以让每个 node 维护"经过这个 node 的所有句子 set"，O(1) 取集合。

🪜 思路拆解

Step 1 · 数据结构 - Trie node: children dict + sentences set。 - sentence_to_hot: dict。 - 维护"当前输入 buffer" cur 和"当前所在 trie node" cur_node。

Step 2 · __init__ - 把每个历史句子插入 trie（沿路径每个 node 的 sentences add 它）。 - 填 sentence_to_hot。

Step 3 · input(c) - 如果 c == '#'：把 cur 加入 trie（or 增加 hot），重置 cur/cur_node，return []。 - 否则：cur += c，cur_node = cur_node.children.get(c)。 - 取 cur_node.sentences，按 (-hot, sentence) 排序，取前 3 个 sentence。 - 如果 cur_node 不存在（之前没出现过该前缀）→ 返回 []。

Step 4 · 排序 key - 主 key：hot 降序 → -hot。 - 副 key：sentence 升序。

⌨️ 逐步实现

class Trie:
    def __init__(self):
        self.children = {}
        self.sentences = set()

class AutocompleteSystem:
    def __init__(self, sentences, times):
        self.root = Trie()
        self.hot = {}
        for s, t in zip(sentences, times):
            self.hot[s] = t
            self._insert(s)
        self.cur = ""
        self.node = self.root
        self.dead = False                          # 当前前缀已不存在

    def _insert(self, s):
        node = self.root
        for c in s:
            node = node.children.setdefault(c, Trie())
            node.sentences.add(s)

    def input(self, c):
        if c == '#':
            self.hot[self.cur] = self.hot.get(self.cur, 0) + 1
            self._insert(self.cur)
            self.cur = ""
            self.node = self.root
            self.dead = False
            return []
        self.cur += c
        if self.dead or c not in self.node.children:
            self.dead = True
            return []
        self.node = self.node.children[c]
        sents = self.node.sentences
        return sorted(sents, key=lambda s: (-self.hot[s], s))[:3]

讲解策略

  1. Trie 节点维护 sentences 集合是关键设计。
  2. dead 标志避免无意义查找。
  3. 排序 key 用 tuple (-hot, s) 一次搞定。
  4. 优化：每节点缓存 top-3 → 进阶讨论。

常见坑

  • # 后状态重置：cur、node、dead 都要清。
  • 新句子可能不在 trie：# 时要 _insert 进去。
  • dead 标志：一旦失配，后续 input 应直接 []，不应再走 trie。
