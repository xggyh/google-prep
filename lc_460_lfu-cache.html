LC 460. LFU Cache LFU 缓存

类别：🏗️ OOP 设计 · 难度：Hard · 标记：➕ 额外推荐

🏗️ OOP 设计 Hard ➕ 额外推荐

题目概览

🏷️ 类别：OOP 设计 / 双向链表 + HashMap | 难度：Hard | 标记：➕

题目

EN: Design a Least Frequently Used (LFU) cache with capacity. Support get and put in O(1). Eviction: lowest frequency; tie → least recently used.

中文：设计 LFU 缓存，支持 O(1) 的 get / put。淘汰策略：频率最低；同频淘汰最久未用。

澄清问题

  • EN: Capacity 0 case? / Update on get (frequency)? / Update on put (existing key)?
  • 中：容量 0 怎么办？get 是否增加频率？put 已存在 key 怎么处理？

🧠 整体思路

3 个数据结构： 1. key_to_val：key → value。 2. key_to_freq：key → 当前频率。 3. freq_to_keys：freq → OrderedDict（保持插入顺序，即"在该频率内的访问顺序"）。

额外维护 min_freq：当前最小频率，淘汰时直接用。

get(key)： - 不存在 → -1。 - 存在 → 把 key 从 freq_to_keys[old_freq] 移到 freq_to_keys[old_freq+1]；若 old_freq 桶空且 old_freq == min_freq → min_freq += 1。

put(key, val)： - 已存在 → 更新 val，同 get 流程。 - 不存在： - 若满 → 淘汰 freq_to_keys[min_freq] 第一个 key。 - 加入新 key，freq = 1，min_freq = 1。

复杂度：O(1) per op（OrderedDict 的 popitem 是 O(1)）。

🪜 思路拆解

Step 1 · 初始化 - 4 个状态：key_to_val、key_to_freq、freq_to_keys (defaultdict(OrderedDict))、min_freq。

Step 2 · 私有方法 _touch(key) - 把 key 在 freq_to_keys 里从 old_freq 移到 old_freq+1。 - 更新 key_to_freq。 - 若旧桶空且是 min_freq → min_freq++。

Step 3 · get - 不存在 → -1。 - 存在 → _touch(key)，返回 val。

Step 4 · put - 容量 0 直接 return。 - 已存在 → 更新 val 后 _touch。 - 不存在： - 若满 → 淘汰 freq_to_keys[min_freq] 的第一个。 - 插入新 key（freq=1，min_freq=1）。

⌨️ 逐步实现

from collections import OrderedDict, defaultdict

class LFUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.size = 0
        self.min_freq = 0
        self.key_to_val = {}
        self.key_to_freq = {}
        self.freq_to_keys = defaultdict(OrderedDict)

_touch 提升频率

    def _touch(self, key):
        f = self.key_to_freq[key]
        del self.freq_to_keys[f][key]
        if not self.freq_to_keys[f]:
            del self.freq_to_keys[f]
            if f == self.min_freq:
                self.min_freq += 1
        self.freq_to_keys[f + 1][key] = None
        self.key_to_freq[key] = f + 1

get

    def get(self, key: int) -> int:
        if key not in self.key_to_val:
            return -1
        self._touch(key)
        return self.key_to_val[key]

put

    def put(self, key: int, value: int) -> None:
        if self.cap == 0:
            return
        if key in self.key_to_val:
            self.key_to_val[key] = value
            self._touch(key)
            return
        if self.size == self.cap:
            evict_key, _ = self.freq_to_keys[self.min_freq].popitem(last=False)
            if not self.freq_to_keys[self.min_freq]:
                del self.freq_to_keys[self.min_freq]
            del self.key_to_val[evict_key]
            del self.key_to_freq[evict_key]
            self.size -= 1
        self.key_to_val[key] = value
        self.key_to_freq[key] = 1
        self.freq_to_keys[1][key] = None
        self.min_freq = 1
        self.size += 1

讲解策略

  1. 3 个 hash 表 + min_freq 是核心。
  2. OrderedDict 保证 O(1) popitem(last=False)。
  3. _touch 抽出来避免 get/put 重复代码。
  4. 复杂度 O(1) per op。

常见坑

  • 容量 0：直接 return，不进任何状态。
  • min_freq 更新时机：旧桶空且是 min_freq 时才 +1。
  • put 已存在 key：要更新 val 并 _touch，不要当淘汰流程。
