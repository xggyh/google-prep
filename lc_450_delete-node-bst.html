LC 450. Delete Node in a BST 删除二叉搜索树中的节点

类别：🌳 树 / BST · 难度：Med · 标记：⭐ Google 直接对应

🌳 树 / BST Med ⭐ Google 直接对应

题目概览

🏷️ 类别：树 / BST | 难度：Medium | 标记：⭐

题目

EN: Given root of BST and a key, delete the node with that key and return the new root.

中文：BST 中删除值为 key 的节点，返回新根。

澄清问题

  • EN: Unique values? / key may not exist? / Modify in place ok?
  • 中：值唯一吗？key 不存在如何处理？可以原地修改吗？

🧠 整体思路

3 种情况： 1. 叶子：直接删。 2. 一个子：用子替换。 3. 两个子：用右子树最左节点（后继）的值覆盖自己，递归删后继。

为什么用后继：保持 BST 性质 — 后继比左子树所有都大、比右子树其他都小。

🪜 思路拆解

Step 1 · 边界 / 二分查找 - 空树 → None。 - key < val → 递归左。 - key > val → 递归右。 - 等 → 进入删除。

Step 2 · 删除：情况 1 & 2 - 无左 → return right。 - 无右 → return left。

Step 3 · 情况 3 - 找右子树最左节点。 - 复制值。 - 递归删除右子树中后继的原节点。

⌨️ 逐步实现

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val, self.left, self.right = val, left, right

def deleteNode(root, key):
    if not root:
        return None
    if key < root.val:
        root.left = deleteNode(root.left, key)
    elif key > root.val:
        root.right = deleteNode(root.right, key)
    else:
        if not root.left:
            return root.right
        if not root.right:
            return root.left
        succ = root.right
        while succ.left:
            succ = succ.left
        root.val = succ.val
        root.right = deleteNode(root.right, succ.val)
    return root

讲解策略

  1. 3 种情况清晰列出。
  2. 后继选择（右子树最左）画图说明。
  3. 复杂度 O(h)。

常见坑

  • 后继找错：是右子树最左节点，不是"右子树最右"。
  • 复制值后忘递归删后继：会出现重复值。
