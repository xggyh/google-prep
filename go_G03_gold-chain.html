Google 原创 G03. Gold Chain Equal Split 金链平分

类别：📐 DP / 数组 · 难度：Med · 标记：🆕 Google 原创

📐 DP / 数组 Med 🆕 Google 原创

题目概览

🏷️ 类别：DP / 数组 | 难度：Medium | 标记：Google 原创 | 来源：NG R2 R1

题目

EN: Given array a[] representing a gold chain where each cell is a segment's weight. Remove one segment, then reconnect the chain (so it's now n-1 segments in a line). Can the remaining chain be split into two contiguous halves with equal sum? Return True/False. Follow-up: return all valid (remove_idx, split_idx) pairs.

中文：金链数组 a[]，每项是一节的重量。移除一节后链条 reconnect 成 n-1 节连续。问能否再把它切成两段连续子数组，两人重量相等。Follow-up：返回所有可行方案。

澄清问题

  • EN: After removing, do remaining items keep order? (Yes — concatenated) / Can the split halve be empty?
  • 中：移除后剩下的元素保持原顺序拼接吗（是的）？两段能为空吗？

🧠 整体思路

核心 = 前缀和 + 枚举：

Total = sum(a)。移除 a[k] 后 remaining = Total - a[k]。must be even 才能平分。

枚举移除的 k： - 移除后 remaining = Total - a[k]，目标每人 = (Total - a[k]) / 2。 - 检查是否存在某 split point j 使得"前段 sum == target"。

优化：用前缀和 + HashSet 快速检查 (k, j) 对。

复杂度：朴素 O(n²)；优化到 O(n)。

🪜 思路拆解

Step 1 · 算前缀和 pref[i] = sum(a[0..i-1])

Step 2 · 枚举移除 k 从 0 到 n-1 - remaining = Total - a[k]，必须为偶数，否则 skip。 - target = remaining / 2。

Step 3 · 枚举 split point j（在 a[k] 移除后的链上） - 前段是 a[0..k-1] + a[k+1..j-1]（如果 j > k），sum = pref[j] - a[k] if j > k else pref[j]。 - 实际上更简洁：前段是移除 k 后的前 ? 个元素。

等价转化：移除 k 后形成长度 n-1 的数组 b。求 b 的某前缀 sum = target。 - 如果 split 在 j' (in b's index)，对应原始数组中： - 若 j' ≤ k-1: 前段就是 a[0..j']，sum = pref[j'+1]。 - 若 j' ≥ k: 前段是 a[0..k-1] + a[k+1..j'+1]（在 b 中索引 j' 对应 a 中索引 j'+1），sum = pref[j'+2] - a[k]。

Step 4 · 用 HashSet 加速 - 一遍预计算"哪些 pref[i] 值存在"。 - 枚举 k 时，要找的 target 值有两个匹配位置（k 之前 / k 之后），各自查 HashSet。

⌨️ 逐步实现

Step 1-2 · 主函数（朴素版 O(n²)）

def can_split(a):
    n = len(a)
    total = sum(a)
    for k in range(n):
        rem = total - a[k]
        if rem % 2 != 0:
            continue
        target = rem // 2
        # 移除 a[k] 后枚举 split point
        cur = 0
        for j in range(n):
            if j == k:
                continue
            cur += a[j]
            if cur == target:
                return True
            if cur > target:
                break
    return False

Follow-up · 返回所有 (k, split) 对

def all_splits(a):
    n = len(a)
    total = sum(a)
    result = []
    for k in range(n):
        rem = total - a[k]
        if rem % 2 != 0:
            continue
        target = rem // 2
        cur = 0
        for j in range(n):
            if j == k:
                continue
            cur += a[j]
            if cur == target:
                result.append((k, j))
                # 不 break, 继续找
    return result

讲解策略

  1. 拆两层：枚举移除 k → 枚举 split point。
  2. 先朴素 O(n²) 跑通，再讨论前缀和 + HashSet O(n)。
  3. 强调 rem % 2 == 0 的提前判断。

常见坑

  • rem 必须偶数：奇数直接 skip，不然 // 2 会向下取整出错。
  • 跳过 k：枚举 split 时不能算 a[k] 进去。
  • break 时机：如果数组全正数，cur > target 后可以早停；含负数不能 break。
