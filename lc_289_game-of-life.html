LC 289. Game of Life 生命游戏

类别：🎮 模拟 / 链表 / 并发 · 难度：Med · 标记：⭐ Google 直接对应

🎮 模拟 / 链表 / 并发 Med ⭐ Google 直接对应

题目概览

🏷️ 类别：模拟 / In-place 编码 | 难度：Medium | 标记：⭐

题目

EN: Conway's Game of Life on m×n board (0=dead, 1=live, 8-neighbor). Update to next generation. Follow-up: in-place.

中文：康威生命游戏，0 死 1 活，8 邻接。更新到下一代。Follow-up：原地。

澄清问题

  • EN: All 8 neighbors? / Sync update? / Multiple steps?
  • 中：8 邻？同步更新？只 1 步？

🧠 整体思路

4 状态编码 in-place： - 0 → 还 0；1 → 还 1； - 2 = 原来 1，新一代要变 0； - 3 = 原来 0，新一代要变 1。

邻居计数时只看 in (1, 2)（原来活）。

两遍扫：标记 → 还原。

🪜 思路拆解

Step 1 · 边界 - 空 board / 空 row 直接 return。

Step 2 · 第一遍：打标记 - 对每格数邻居（邻居在 (1, 2) 才算活）。 - 1 → 邻居 <2 or >3 → 标 2。 - 0 → 邻居 ==3 → 标 3。

Step 3 · 第二遍：还原 - 2 → 0；3 → 1。

⌨️ 逐步实现

def gameOfLife(board):
    if not board or not board[0]:
        return
    m, n = len(board), len(board[0])

    def count(i, j):
        c = 0
        for di in (-1, 0, 1):
            for dj in (-1, 0, 1):
                if di == 0 and dj == 0:
                    continue
                ni, nj = i + di, j + dj
                if 0 <= ni < m and 0 <= nj < n and board[ni][nj] in (1, 2):
                    c += 1
        return c

    for i in range(m):
        for j in range(n):
            live = count(i, j)
            if board[i][j] == 1 and (live < 2 or live > 3):
                board[i][j] = 2
            elif board[i][j] == 0 and live == 3:
                board[i][j] = 3

    for i in range(m):
        for j in range(n):
            board[i][j] = 1 if board[i][j] in (1, 3) else 0

讲解策略

  1. 先 O(mn) 空间版（复制），再 in-place follow-up。
  2. 状态编码 2/3 是关键技巧。
  3. Follow-up：无限大棋盘 → 只存活细胞坐标 set。

常见坑

  • 同步更新理解错：邻居要看原始状态。
  • state ∈ {1, 2} 算活：第一遍标记后还要看"原来"是不是活。
  • 边界检查：双 for 邻居要判越界。
