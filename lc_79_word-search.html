LC 79. Word Search 单词搜索

类别：🗺️ 图 / BFS / DFS · 难度：Med · 标记：🔄 LC 变体

🗺️ 图 / BFS / DFS Med 🔄 LC 变体

题目概览

🏷️ 类别：图 / DFS+Backtrack | 难度：Medium | 标记：🔄

题目

EN: Given an m x n grid of characters board and a string word, return true if word exists in the grid. The word can be constructed from letters of sequentially adjacent cells (horizontally or vertically), where the same cell may not be used more than once.

中文：在 m×n 字符网格中查找单词。单词由相邻（上下左右）格子组成，同一格子不能重复使用。

澄清问题

  • EN: Case-sensitive? / Grid size? / Word length? / Can the word be empty? / Are there special characters?
  • 中：大小写敏感？网格规模？单词长度？单词可以为空吗？有特殊字符吗？

🧠 整体思路

DFS + 回溯。从每个可能的起点（字符 = word[0]）出发，沿 4 方向搜索，匹配 word 的下一个字符。

关键技巧 — visited 标记直接改 board：把已访问的格子临时改成特殊字符（如 #），递归返回时改回来。省了一个 visited 集合的空间。

复杂度：O(m·n · 4^L)，L 是 word 长度。最坏指数级，实际剪枝后很快。

🪜 思路拆解

Step 1 · 边界 - 空 board / 空 word：早返回。

Step 2 · 外层循环：枚举起点 - 遍历所有格子，字符 = word[0] 的尝试 DFS。

Step 3 · DFS 函数签名 - dfs(i, j, idx)：当前在 (i, j)，匹配 word[idx]。 - 返回 True 表示能完成 word[idx:] 的匹配。

Step 4 · DFS 终止与剪枝 - 越界 / 字符不匹配 → False。 - idx == len(word) - 1 且字符匹配 → True。

Step 5 · 标记 + 递归 + 回溯 - 改 board[i][j] = '#'（标记 visited）。 - 4 方向递归，任一 True 就 True。 - 改回 board[i][j] = word[idx]（无论成败都要恢复）。

Python 完整实现

def exist(board, word):
    if not word:
        return True
    m, n = len(board), len(board[0])

    def dfs(i, j, idx):
        if idx == len(word):
            return True
        if i < 0 or i >= m or j < 0 or j >= n or board[i][j] != word[idx]:
            return False
        ch = board[i][j]
        board[i][j] = '#'
        found = (dfs(i+1, j, idx+1) or dfs(i-1, j, idx+1) or
                 dfs(i, j+1, idx+1) or dfs(i, j-1, idx+1))
        board[i][j] = ch
        return found

    for i in range(m):
        for j in range(n):
            if board[i][j] == word[0] and dfs(i, j, 0):
                return True
    return False

讲解策略

  1. 先讲思路：DFS + 回溯，强调"标记 visited 直接改 board"省空间。
  2. 时间复杂度 O(m·n · 4^L) 主动给。
  3. 提优化：起点剪枝（先比较 board 中 word[0] 和 word[-1] 的频率，从频率小的一端开始搜）。
  4. dry run 一个 3×3 小例子。

常见坑

  • 回溯漏恢复：找到答案直接 return 时也要先恢复 board。
  • visited 没标记：同一格子被重复访问导致死循环。
  • idx 越界：终止条件要先判 idx == len(word) 再判字符匹配。
  • 修改 board 副作用：面试场景一般允许，但要主动说明。
