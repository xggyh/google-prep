Google 原创 G01. File System Total Size 文件系统总大小（带 cycle / cache）

类别：🌳 树 / BST · 难度：Med · 标记：🆕 Google 原创

🌳 树 / BST Med 🆕 Google 原创

题目概览

🏷️ 类别：树 / 递归 + 缓存 | 难度：Medium | 标记：Google 原创 | 来源：26ng r2 onsite R1

题目

EN: Compute the total size of a directory in a file system. Each file has a size; each directory contains files + sub-directories. Compute recursively. Variants: (1) hard links may create cycles, need cycle detection; (2) optimize repeated calls with cache.

中文：计算文件系统某目录的总大小。递归求子文件 + 子目录。变体：(1) hard link 可能成环，需要 cycle detection；(2) 多次查询，要用 cache 优化。

澄清问题

  • EN: Hard links possible? / Are sizes pre-computed for files only? / Multi-threaded calls?
  • 中：有 hard link（环）吗？只有 file 有 size 还是 dir 也有？多次查询？

🧠 整体思路

最基础版本（严格树）： - 递归：dir 的 size = 所有子文件 size + 所有子目录递归 size。 - O(N)。

变体 1：cycle detection： - 用 visited set 避免重复访问。

变体 2：cache（多次查询场景）： - 给每个 dir 缓存 size，第二次访问直接返回。 - 注意：cache invalidation 是难点 — 文件添加/删除后要重新算。

复杂度：第一次 O(N)，后续命中 O(1)。

🪜 思路拆解

Step 1 · 数据结构假设 - dir.subdirs: list[Dir]，dir.files: list[File]，file.size: int。

Step 2 · 朴素递归 - total(dir) = sum(f.size for f in dir.files) + sum(total(sd) for sd in dir.subdirs)。

Step 3 · cycle detection（变体 1） - 维护 visited set，遇到已访问的 dir 跳过。

Step 4 · cache（变体 2） - 在 Dir 对象上加 _cached_size（懒计算）。 - 第一次访问算出来 + 存；之后直接返回。 - 修改时清理：上游每个父 dir 的 cache 也要 invalidate。

⌨️ 逐步实现

Step 2 · 基础递归

def total_size(dir):
    return sum(f.size for f in dir.files) + sum(total_size(sd) for sd in dir.subdirs)

Step 3 · cycle detection

def total_size_cyclesafe(dir):
    visited = set()
    def dfs(d):
        if id(d) in visited:
            return 0
        visited.add(id(d))
        return sum(f.size for f in d.files) + sum(dfs(sd) for sd in d.subdirs)
    return dfs(dir)

Step 4 · cache 版

class Dir:
    def __init__(self):
        self.files = []
        self.subdirs = []
        self.parent = None
        self._cached_size = None

    def size(self):
        if self._cached_size is not None:
            return self._cached_size
        s = sum(f.size for f in self.files) + sum(sd.size() for sd in self.subdirs)
        self._cached_size = s
        return s

    def invalidate(self):
        # 当 self 内容变化时, 清掉自己和所有祖先的 cache
        node = self
        while node is not None:
            node._cached_size = None
            node = node.parent

讲解策略

  1. 先做最简单 O(N) 朴素版。
  2. 主动提 cycle 风险（hard link）→ visited set。
  3. follow-up 多次查询 → cache + invalidation。
  4. 复杂度 first call O(N)，subsequent O(1)（命中）。

常见坑

  • 递归用 id(d) 而非对象本身：避免 hashable 问题。
  • cache invalidation：修改文件时，所有祖先的 cache 都要清，不只当前 dir。
  • "strict tree" 简化：如果面试官说严格树，可以省 visited，但 cache 仍可加。
