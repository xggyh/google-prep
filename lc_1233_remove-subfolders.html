LC 1233. Remove Sub-Folders from the Filesystem 删除子文件夹

类别：🔤 字符串 / 解析 / 栈 · 难度：Med · 标记：🔄 LC 变体

🔤 字符串 / 解析 / 栈 Med 🔄 LC 变体

题目概览

🏷️ 类别：字符串 / 排序 | 难度：Medium | 标记：🔄

题目

EN: Given a list of folders, return folders without any sub-folder. A sub-folder is one whose path starts with another folder's path + /.

中文：给一组文件夹路径，去掉所有"子文件夹"（路径以另一个文件夹路径 + / 开头）。

澄清问题

  • EN: Are paths Unix-style (/a/b)? / Empty path? / Sorted?
  • 中：路径格式（Unix 风格 /a/b）？空路径？已排序？

🧠 整体思路

排序后线性扫描： - 字典序排序后，父文件夹一定在子文件夹之前。 - 维护"上一个保留的父" prev。 - 当前路径以 prev + '/' 开头 → 是子文件夹，跳过。 - 否则保留 + 更新 prev。

复杂度：O(n log n · L)，L 是平均路径长度。

🪜 思路拆解

Step 1 · 排序

Step 2 · 线性扫描 - 初始 prev = None。 - 对每个 folder： - 若 prev 是 None 或 folder 不以 prev + '/' 开头 → 保留 + 更新 prev。

⌨️ 逐步实现

def removeSubfolders(folders):
    folders.sort()
    result = []
    for f in folders:
        if not result or not f.startswith(result[-1] + '/'):
            result.append(f)
    return result

讲解策略

  1. 关键洞察：排序后父在子前。
  2. 一行 startswith 判定关系。
  3. 复杂度 O(n log n · L)。
  4. 提替代：trie 也行，但排序更简洁。

常见坑

  • 没加 /：/a 不应被视为 /ab 的父 — 必须 prev + '/'。
  • 空 folders：return []。
