Google 原创 G13. Top-K Logger 频次日志 Top K

类别：📚 堆 / 扫描线 · 难度：Med · 标记：🆕 Google 原创

📚 堆 / 扫描线 Med 🆕 Google 原创

题目概览

🏷️ 类别：OOP 设计 / 堆 + HashMap | 难度：Medium | 标记：Google 原创 | 来源：L3/L4 编译 #33

题目

EN: In-memory logger: log(event), topK(k) returns top k most frequent events.

中文：日志器，支持 log(event)、topK(k) 返回 top k 频次事件。

🧠 整体思路

HashMap + heap-on-demand： - freq: Counter。 - topK 查询时用 heapq.nlargest(k, freq.items(), key=lambda x: x[1])。

进阶：维护实时 top-k 堆（每次 log 更新）— 但需处理 stale。

🪜 思路拆解（简单版）

Step 1 · log: freq[event] += 1 Step 2 · topK: heapq.nlargest

⌨️ 逐步实现

import heapq
from collections import Counter

class Logger:
    def __init__(self):
        self.freq = Counter()

    def log(self, event):
        self.freq[event] += 1

    def topK(self, k):
        return [e for e, _ in heapq.nlargest(k, self.freq.items(), key=lambda x: x[1])]

讲解策略

  1. 朴素版：HashMap + heapq.nlargest。
  2. 进阶：维护实时 top-k → 用 sorted structure（红黑树 / TreeMap）。
  3. Trade-off：log 慢 vs topK 慢。

常见坑

  • 频次相同打破平局：先问。
  • 大量 log + 偶尔 topK → 朴素版好。
