LC 207. Course Schedule 课程表（feature 依赖激活）

类别：🗺️ 图 / BFS / DFS · 难度：Med · 标记：🔄 LC 变体

🗺️ 图 / BFS / DFS Med 🔄 LC 变体

题目概览

🏷️ 类别：图 / 拓扑排序 | 难度：Medium | 标记：🔄

题目

EN: Given numCourses and prerequisites [a, b] (take b before a). Return True if you can finish all courses.

中文：n 门课加先修关系 [a, b] 表示学 a 前要学 b。能学完所有课吗？

0522 面经里的"feature 依赖激活"就是这道题的变体 — feature 间有依赖，必须激活前置 feature 才能激活新的。

澄清问题

  • EN: Can there be duplicate edges? / Self-loops? / Disconnected components OK?
  • 中：重边？自环？图允许不连通吗？

🧠 整体思路

判断有向图是否有环。两种解法：

  1. Kahn 算法（BFS 拓扑排序）：入度为 0 的入队，反复 pop 并消邻居的入度。最后处理过的数 < n → 有环。
  2. DFS + 三色标记：白 / 灰 / 黑。访问到灰 → 有环。

Kahn 更直观且容易写对。

复杂度：O(V + E)。

🪜 思路拆解（Kahn 版）

Step 1 · 建图 + 入度数组 - graph[b] = [a, ...]（b 完成后可以让 a 解锁）。 - indegree[a] += 1 对每个 prerequisite。

Step 2 · 把入度 0 的入队

Step 3 · BFS - pop 一个 → 处理计数 +1。 - 遍历它的邻居，邻居入度 -1，归零就入队。

Step 4 · 判断 - 处理过的数 == n → 无环 / 可以完成。

⌨️ 逐步实现

from collections import defaultdict, deque

def canFinish(numCourses, prerequisites):
    graph = defaultdict(list)
    indegree = [0] * numCourses
    for a, b in prerequisites:
        graph[b].append(a)
        indegree[a] += 1

    queue = deque(i for i in range(numCourses) if indegree[i] == 0)
    done = 0
    while queue:
        u = queue.popleft()
        done += 1
        for v in graph[u]:
            indegree[v] -= 1
            if indegree[v] == 0:
                queue.append(v)
    return done == numCourses

讲解策略

  1. 抽象成"有向图判环"。
  2. Kahn 算法的核心 idiom 是"入度为 0 才入队"。
  3. 复杂度 O(V + E)。
  4. follow-up 可以变 LC 210 返回拓扑序、LC 269 火星词典等。

常见坑

  • 建反图：[a, b] 是"学 a 前学 b"，所以 b → a，不要写反。
  • 重边：题目通常不保证无重，多次 indegree += 1 会出错。先 dedup 比较稳。
  • 不连通组件：每个独立连通块都要从 indegree=0 开始 — Kahn 自动处理。
