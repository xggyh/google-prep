Q10. 进出门时间表（带优先级 tie-breaking）

来源：SDE NG VO R2 题2 · LC: 无直接 LC ❓ 无对应 · 难度: Hard

Queue Heap Simulation ❓ 无对应 Hard

题目概览

来源：Google SDE NG VO R2 第二题 原描述：有序数组，每个索引代表不同的人，元素是 (timestamp, "进"/"出")。返回每人的实际进/出时间。tie-breaking：前一刻进 → 进优先；前一刻出 → 出优先；前一刻无 → 出优先；同时进/出 → 索引小的优先。

题目

EN: An array indexed by person ID, where arr[i] = (timestamp, "in"|"out") indicates person i wants to enter or exit at time timestamp. At each tick only one person can pass through the door. Resolve schedule with rules: - If previous action was "in" → "in" requests have priority. - If previous was "out" → "out" has priority. - If door was idle → "out" priority. - Within same direction, lower index goes first. Return for each person the actual time they enter/exit.

中文：（同原描述）

澄清问题

  • EN: One person per timestamp unit? / Time is integer or real? / Can a person have multiple requests? / What if no one is queued? door stays in last state? / What's "previous action" if it was a tie? / Output format?
  • 中：每个时刻只能过一个人吗？时间是整数还是实数？同一个人能有多个请求吗？没人时门保持上一个状态还是 reset？如果上一刻同时有人进出（tie），算"进"还是"出"？

🧠 整体思路

这题是模拟题，但 tie-breaking 规则多到容易写错。先把规则完整地写在纸上再动手。

模型：两个队列分别管"想进门的人"和"想出门的人"，按时间推进。每个时间单位最多一个人通过门。

4 条 tie 规则（必须全部覆盖）： 1. 上一刻进 → 进优先 2. 上一刻出 → 出优先 3. 上一刻无人 → 出优先（默认） 4. 同方向中 → 索引小的先

朴素 O(T) 是"时间一点点加"；优化 O(n log n) 是"跳到下一个有人的时刻"。

关键状态：prev（上一次通过门的方向：in / out / none）+ cur_time。

🪜 思路拆解

Step 1 · 输入分桶 - 遍历 requests，按 action 把 (timestamp, person_id) 分到 in_heap 或 out_heap。 - 用堆是因为后面要按 timestamp 升序 pop。

Step 2 · 主循环结构 - 终止：两个堆都空。 - 每轮： - 找当前可处理的时间（max(cur_time, 两堆最早时间)）。 - 判断当前时刻哪些"在场" — 即 timestamp ≤ cur_time。 - 应用 4 条 tie 规则选一个方向。 - pop 出对应人，记录 result[pid] = cur_time。 - 更新 prev 和 cur_time += 1。

Step 3 · "在场" 判定 - 用 heap[0][0] <= cur_time 判断堆顶是否在场。 - 注意 cur_time 是动态推进的。

Step 4 · 应用 tie 规则 - 双方都在场： - prev == "in" → 选 in 队（规则 1）。 - prev == "out" → 选 out 队（规则 2）。 - prev == "none" → 选 out 队（规则 3）。 - 一方在场：选那一方。 - 都不在场：跳到下一个有人的时刻。

Step 5 · 同方向多人 → 索引小的先 - 堆的元素是 (timestamp, person_id)，相同 timestamp 时 Python 自动比 id，小的先 pop。所以堆的元组顺序很关键。

Step 6 · prev 初始化 - 一开始没人通过 → prev = "out"（规则 3：默认出优先）或 prev = "none" 然后在判定里映射成 out。 - 题目说"前一刻没人，出门优先"，所以等价于 prev = out。

Python 完整实现

from collections import deque


def schedule(requests):
    n = len(requests)
    # 进队列和出队列，存 (timestamp, person_id)
    in_q = deque()
    out_q = deque()
    for i, (t, action) in enumerate(requests):
        if action == "in":
            in_q.append((t, i))
        else:
            out_q.append((t, i))
    # 按 timestamp 排序（题目说有序，但 defensive）
    in_q = deque(sorted(in_q))
    out_q = deque(sorted(out_q))

    result = [0] * n
    prev = "none"  # "in" / "out" / "none"
    cur_time = 0

    while in_q or out_q:
        # 跳到下一个有人的时间点
        next_t = min(in_q[0][0] if in_q else float('inf'),
                     out_q[0][0] if out_q else float('inf'))
        cur_time = max(cur_time, next_t)

        # 这一时刻可用的人
        in_avail = [pid for t, pid in in_q if t <= cur_time]
        out_avail = [pid for t, pid in out_q if t <= cur_time]

        if in_avail and out_avail:
            chosen_dir = "in" if prev == "in" else "out"
        elif in_avail:
            chosen_dir = "in"
        else:
            chosen_dir = "out"

        if chosen_dir == "in":
            in_avail.sort()
            pid = in_avail[0]
            in_q = deque((t, p) for t, p in in_q if p != pid)
        else:
            out_avail.sort()
            pid = out_avail[0]
            out_q = deque((t, p) for t, p in out_q if p != pid)

        result[pid] = cur_time
        prev = chosen_dir
        cur_time += 1  # 每人占用 1 个时间单位
    return result

⚠️ 上面是清晰版（O(n²)）。优化版用堆，复杂度 O(n log n)。

优化版（堆）

import heapq


def schedule_opt(requests):
    n = len(requests)
    in_heap = []
    out_heap = []
    for i, (t, action) in enumerate(requests):
        if action == "in":
            heapq.heappush(in_heap, (t, i))
        else:
            heapq.heappush(out_heap, (t, i))

    result = [0] * n
    prev = "out"  # 初始时按 "如果前一刻没人，出门优先"
    cur_time = 0

    def avail(heap):
        return heap and heap[0][0] <= cur_time

    while in_heap or out_heap:
        if not (avail(in_heap) or avail(out_heap)):
            cur_time = min(in_heap[0][0] if in_heap else float('inf'),
                           out_heap[0][0] if out_heap else float('inf'))
        if avail(in_heap) and avail(out_heap):
            chosen = in_heap if prev == "in" else out_heap
            prev = "in" if chosen is in_heap else "out"
        elif avail(in_heap):
            chosen = in_heap
            prev = "in"
        else:
            chosen = out_heap
            prev = "out"
        _, pid = heapq.heappop(chosen)
        result[pid] = cur_time
        cur_time += 1
    return result

讲解策略

  1. 先复述规则：用 3-4 个具体例子让面试官确认 tie-breaking 你理解对了。
  2. 两个队列建模：进队 + 出队是 obvious data structure。
  3. 朴素法：时间一点点加（题目原话提到），O(T)，T 是最大时间戳。
  4. 优化：直接跳到"下一个有人"的时间点，O(n log n)。
  5. 强调 prev_state 的维护：这题最容易出 bug 的地方。
  6. 画时间轴：在白板/doc 上画时间轴 + 谁在哪个队列，让面试官跟得上。

常见坑

  • tie-breaking 漏一条：4 条规则要全部覆盖，少一条就错。
  • "同时"vs"同 timestamp"：要确认是同一时刻就 tie，还是要求同时到达。
  • person 可能被处理多次：输入按"每人一条记录"理解，但要问清楚。
  • prev 初始值：题目说"前一刻没人，出门优先"，所以 prev = "none" 时按 out 处理。
