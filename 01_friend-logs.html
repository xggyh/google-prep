Q01. 朋友圈与解除好友（LC 1101 The Earliest Moment When Everyone Become Friends）

来源：L4 AI/ML 四轮 R3 · LC: LC 1101 ✅ 高置信 · 难度: Med

Union-Find BFS Graph ✅ 高置信 Med

题目概览

来源：Google L4 AI/ML 四轮全套面经 R3 原描述：LC 妖妖灵妖（1101）「找朋友」，follow up：如果也有 unfriend 怎么做？建图 + BFS。

题目

EN: You are given logs [timestamp, user_a, user_b] representing friendship events. Two people become friends, and friendship is transitive (friend's friend is a friend). Return the earliest timestamp when everyone in [0, n-1] is in one friend group, or -1.

Follow-up: What if logs also contain unfriend events? Find the earliest timestamp everyone is connected.

中文：给定一组日志 [时间戳, 用户A, 用户B] 代表加好友事件，好友关系传递。返回所有 n 个人成为一个朋友圈的最早时间，找不到返回 -1。Follow-up：如果还有解除好友 (unfriend) 操作呢？

澄清问题

  • EN: Are logs sorted by timestamp? / Is friendship symmetric (a-b ⇔ b-a)? / Can the same pair friend each other twice? / What's the range of n? / For unfriend: can you unfriend if not currently friends?
  • 中：日志是否按时间排序？好友关系是否对称？同一对人会被加好友多次吗？n 的范围？unfriend 时如果本来就不是好友怎么处理？unfriend 之后是要重新算连通分量吗？

🧠 整体思路

这道题在考三件事： 1. 抽象能力：能不能把"朋友圈"这个生活语言抽象成"图的连通分量"。 2. 数据结构选型：动态合并 + 快速查询"是否同组" = Union-Find (DSU)。 3. 何时停止：连通分量数 = 1 就是答案，不用等所有事件。

核心 insight：友谊只增不减 ⇒ 连通分量数单调递减。维护一个 components 计数器，每次成功合并就 -= 1，到 1 时立刻返回 t。

为什么不用 BFS：每次新加边后跑一次 BFS 检查全连通，单次 O(N+E)，总 O(E·(N+E))。Union-Find 把单次降到 O(α(N)) ≈ O(1)，总 O(E·α(N))。

复杂度目标：时间 O(E log E + E·α(N))（排序主导），空间 O(N)。

🪜 思路拆解（Step-by-Step）

Step 1 · 排序事件 - 做什么：把 logs = [[t, a, b], ...] 按 t 升序排。 - 为什么：要找最早时间，必须按时间顺序处理。 - 用什么：logs.sort(key=lambda x: x[0])。 - 注意：先问面试官 "logs 是否已排序"，没排序你来排。

Step 2 · 设计 Union-Find 的状态 - 做什么：维护三个字段。 - parent[i]：节点 i 的父节点（初始指向自己）。 - rank[i]：以 i 为根的树的近似高度（初始 0），用于按秩合并。 - components：当前连通分量数（初始 n），核心计数器。 - 为什么 rank：让矮树挂到高树下，避免树退化成链表。 - 为什么 components：O(1) 判断"是否全连通"，比每次跑 BFS 快太多。

Step 3 · 实现 find(x) — 找根 + 路径压缩 - 做什么：沿着 parent 链向上爬，直到找到根（parent[x] == x）。 - 为什么压缩：不压缩会退化到 O(n)；压缩后均摊 ≈ O(1)。 - 压缩技巧：用 path halving — 每次把 parent[x] 重新指向 parent[parent[x]]（爷爷）。 - 推荐迭代：避免递归在大 N 时爆栈。

Step 4 · 实现 union(x, y) — 按 rank 合并 - 做什么：把 x 和 y 所在的两棵树合并成一棵。 - 关键判断：先看是不是同根，是就 return False（绝对不能漏！否则 components 计数会错）。 - 怎么合并：让 rank 小的根挂到 rank 大的根下。 - 副作用：合并成功 → components -= 1，return True。

Step 5 · 主循环 — 边处理边检查 - 做什么：按时间顺序遍历每个 (t, a, b)，调 union(a, b)，再看 components == 1。 - 为什么检查每次后：早停就是答案 — 一旦全连通，当前 t 就是最早时间。

Step 6 · 处理无解 - 做什么：循环结束 components 还 > 1，返回 -1。 - 为什么：所有事件都用完了还没把所有人连起来。

⌨️ 逐步实现（Step-by-Step Code）

Step 2 · 初始化 Union-Find 的状态

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))   # 每人初始独立: parent[i] = i
        self.rank = [0] * n            # 树高度近似值
        self.components = n            # 关键: 当前连通分量计数

Step 3 · 找根 + 路径压缩

    def find(self, x):
        # 迭代向上爬, 同时把 x 指向爷爷 (path halving)
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

Step 4 · 按 rank 合并

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False                # 同根, 不算一次合并
        # 矮树挂高树下
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        self.components -= 1            # 关键: 计数 -1
        return True

Step 1 + 5 + 6 · 主函数

def earliest_acquaintance(logs, n):
    logs.sort(key=lambda x: x[0])       # Step 1
    uf = UnionFind(n)
    for t, a, b in logs:                # Step 5
        uf.union(a, b)
        if uf.components == 1:
            return t
    return -1                           # Step 6

Python 完整实现（主问题：Union-Find）

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.components = n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        self.components -= 1
        return True


def earliest_acquaintance(logs, n):
    logs.sort(key=lambda x: x[0])
    uf = UnionFind(n)
    for t, a, b in logs:
        uf.union(a, b)
        if uf.components == 1:
            return t
    return -1

Follow-up（带 unfriend）：建图 + BFS

from collections import defaultdict, deque


def earliest_all_connected(events, n):
    graph = defaultdict(set)
    for t, op, a, b in sorted(events):
        if op == "friend":
            graph[a].add(b)
            graph[b].add(a)
        else:
            graph[a].discard(b)
            graph[b].discard(a)
        if _is_connected(graph, n):
            return t
    return -1


def _is_connected(graph, n):
    visited = {0}
    queue = deque([0])
    while queue:
        u = queue.popleft()
        for v in graph[u]:
            if v not in visited:
                visited.add(v)
                queue.append(v)
    return len(visited) == n

讲解策略（拿 Strong Hire）

  1. 先给 brute force：每次事件都跑一遍 BFS 检查连通性，O(E·(V+E))。
  2. 提出 Union-Find 优化：友谊只增不减时，DSU 是最优解。讲清楚为什么 components 计数能 O(1) 判断是否全连通。
  3. 复杂度：O(E·α(N))，α 是 inverse Ackermann，近似 O(E)。
  4. Follow-up 自然过渡：解释 Union-Find 不支持高效"分裂"，所以 unfriend 必须退化到图 + 每步 BFS（或者维护动态连通性 — 比如 Link-Cut Tree，但面试不必写）。
  5. 主动讨论：如果 unfriend 很稀少，可以"懒删除" + 周期性重建 DSU；如果非常频繁，每步 O(V+E) 也可以接受。

常见坑

  • 忘记排序 logs：题目没明说就要问。
  • Union-Find 写 recursive find：在 N 很大时可能爆栈，迭代写法更稳。
  • 不维护 components 计数：每次都遍历检查会从 O(N·α) 退化到 O(N²·α)。
  • Follow-up 直接说"用 Union-Find 处理 unfriend"：DSU 不能高效 undo，会被面试官追问到死。
