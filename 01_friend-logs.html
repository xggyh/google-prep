01. 朋友圈与解除好友（LC 1101 The Earliest Moment When Everyone Become Friends）

来源：L4 AI/ML 四轮 R3 · LC: LC 1101 ✅ 高置信 · 难度: Med

Union-Find BFS Graph ✅ 高置信 Med

题目概览

来源：Google L4 AI/ML 四轮全套面经 R3 原描述：LC 妖妖灵妖（1101）「找朋友」，follow up：如果也有 unfriend 怎么做？建图 + BFS。

题目

EN: You are given logs [timestamp, user_a, user_b] representing friendship events. Two people become friends, and friendship is transitive (friend's friend is a friend). Return the earliest timestamp when everyone in [0, n-1] is in one friend group, or -1.

Follow-up: What if logs also contain unfriend events? Find the earliest timestamp everyone is connected.

中文：给定一组日志 [时间戳, 用户A, 用户B] 代表加好友事件，好友关系传递。返回所有 n 个人成为一个朋友圈的最早时间，找不到返回 -1。Follow-up：如果还有解除好友 (unfriend) 操作呢？

澄清问题

  • EN: Are logs sorted by timestamp? / Is friendship symmetric (a-b ⇔ b-a)? / Can the same pair friend each other twice? / What's the range of n? / For unfriend: can you unfriend if not currently friends?
  • 中：日志是否按时间排序？好友关系是否对称？同一对人会被加好友多次吗？n 的范围？unfriend 时如果本来就不是好友怎么处理？unfriend 之后是要重新算连通分量吗？

Python 实现（主问题：Union-Find）

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.components = n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        self.components -= 1
        return True


def earliest_acquaintance(logs, n):
    logs.sort(key=lambda x: x[0])
    uf = UnionFind(n)
    for t, a, b in logs:
        uf.union(a, b)
        if uf.components == 1:
            return t
    return -1

Follow-up（带 unfriend）：建图 + BFS

from collections import defaultdict, deque


def earliest_all_connected(events, n):
    graph = defaultdict(set)
    for t, op, a, b in sorted(events):
        if op == "friend":
            graph[a].add(b)
            graph[b].add(a)
        else:
            graph[a].discard(b)
            graph[b].discard(a)
        if _is_connected(graph, n):
            return t
    return -1


def _is_connected(graph, n):
    visited = {0}
    queue = deque([0])
    while queue:
        u = queue.popleft()
        for v in graph[u]:
            if v not in visited:
                visited.add(v)
                queue.append(v)
    return len(visited) == n

讲解策略（拿 Strong Hire）

  1. 先给 brute force：每次事件都跑一遍 BFS 检查连通性，O(E·(V+E))。
  2. 提出 Union-Find 优化：友谊只增不减时，DSU 是最优解。讲清楚为什么 components 计数能 O(1) 判断是否全连通。
  3. 复杂度：O(E·α(N))，α 是 inverse Ackermann，近似 O(E)。
  4. Follow-up 自然过渡：解释 Union-Find 不支持高效"分裂"，所以 unfriend 必须退化到图 + 每步 BFS（或者维护动态连通性 — 比如 Link-Cut Tree，但面试不必写）。
  5. 主动讨论：如果 unfriend 很稀少，可以"懒删除" + 周期性重建 DSU；如果非常频繁，每步 O(V+E) 也可以接受。

常见坑

  • 忘记排序 logs：题目没明说就要问。
  • Union-Find 写 recursive find：在 N 很大时可能爆栈，迭代写法更稳。
  • 不维护 components 计数：每次都遍历检查会从 O(N·α) 退化到 O(N²·α)。
  • Follow-up 直接说"用 Union-Find 处理 unfriend"：DSU 不能高效 undo，会被面试官追问到死。
