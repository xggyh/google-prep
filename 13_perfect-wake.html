13. Perfect Wake Number 计数（数位 DP）

来源：Coding R1 · LC: 数位 DP ❓ 无对应 · 难度: Hard

Digit DP Bitmask Memoization ❓ 无对应 Hard

题目概览

来源：Google coding R1 原描述：定义 Perfect Wake Number：(1) 各位数字不重复，(2) 不含 0，(3) 没有一位数字同时邻接两个比它大的数字（i.e., 不是局部最小，除了端点）。Follow-up：1 到 n 之间有多少个？

题目

EN: A positive integer is a Perfect Wake Number if: 1. No repeated digits. 2. Does not contain digit 0. 3. No digit is adjacent to two strictly larger digits (i.e., no "valley" — every internal digit is ≥ at least one neighbor). Examples: 196, 23, 12463 are valid. 1546 is not (4 has 5 and 6 both larger). 320 invalid (has 0). 34321 invalid (3 repeats).

Follow-up: Count Perfect Wake Numbers in [1, n].

中文：（同上）

澄清问题

  • EN: Is single-digit number always valid? / Can leading digit be anything 1-9? / "Strictly larger" — what about equal? (Moot since no repeats) / n's range — fits in int64? / What about n's number of digits — up to 18?
  • 中：单个数字算 Perfect Wake 吗？n 的范围（决定数位 DP 状态数）？前导是否一定非零（题目说不含 0 所以自然）？是否要返回数字本身还是 count？

检验单个数字 — 简单题

def is_perfect_wake(x):
    s = str(x)
    if '0' in s:
        return False
    if len(set(s)) != len(s):
        return False
    for i in range(1, len(s) - 1):
        if s[i] < s[i-1] and s[i] < s[i+1]:
            return False
    return True

Follow-up：1~n 内 PWN 计数（数位 DP）

def count_perfect_wake(n):
    if n < 1:
        return 0
    digits = list(map(int, str(n)))
    L = len(digits)
    # state: (pos, prev_digit, prev_prev_digit, used_mask, is_tight, started)
    # prev_digit = -1 表示还没开始
    from functools import lru_cache

    @lru_cache(maxsize=None)
    def dp(pos, prev, prev_prev, used_mask, is_tight, started):
        if pos == L:
            return 1 if started else 0
        upper = digits[pos] if is_tight else 9
        total = 0
        # 选择不开始（依然 leading 0）
        if not started:
            total += dp(pos + 1, -1, -1, 0, is_tight and digits[pos] == 0, False)
            start_d = 1
        else:
            start_d = 0  # 但因为不含 0，所以下面统一从 1 开始
        for d in range(max(1, start_d), upper + 1):
            if used_mask & (1 << d):
                continue
            # 检查 valley 规则：prev 不能比 prev_prev 和 d 都小
            if prev != -1 and prev_prev != -1:
                if prev < prev_prev and prev < d:
                    continue  # 这一步如果选 d，会让 prev 变成 valley
            new_tight = is_tight and (d == digits[pos])
            total += dp(pos + 1, d, prev, used_mask | (1 << d), new_tight, True)
        return total

    return dp(0, -1, -1, 0, True, False)

讲解策略

  1. 第一题（单个数字判定）几乎送分，30 秒讲完。
  2. 进入 follow-up 前，明确告诉面试官："这是数位 DP，状态包括位置、是否 tight、是否已开始、前两位"。
  3. 状态设计： - pos: 当前位 - prev, prev_prev: 前两位的数字（用来检查中间的"valley"） - used_mask: 已用数字 bitmask (10 bit) — 因为不重复 - is_tight: 当前位是否受 n 限制 - started: 是否已写出第一位（处理前导 0）
  4. 复杂度：状态数 ≈ L × 10 × 10 × 1024 × 2 × 2 ≈ 800K，可接受。
  5. 画图：在白板上画 dp(0, ...) → dp(1, ...) 的转移，让面试官跟得上。
  6. 诚实：这题在面试 30-45 分钟内全写对很难，能讲清楚 + 写出主体框架 + 1 个例子 trace 就够 hire。

常见坑

  • 忽略前导 0 处理：数位 DP 经典坑，要专门加 started 状态。
  • valley 检查时机错：必须在"选定下一位 d 之后回头检查 prev"，而不是在选 prev 当时。
  • used_mask 忘记：题目要求 no repeated digits。
  • bitmask 用 0 位：因为不含 0，bit 0 永远不会被设置，可以省一位或不省。
  • lru_cache + 全局状态：每次新 n 要清缓存。
