LC 354. Russian Doll Envelopes 俄罗斯套娃信封

类别：📐 DP / 数组 · 难度：Hard · 标记：⭐ Google 直接对应

📐 DP / 数组 Hard ⭐ Google 直接对应

题目概览

🏷️ 类别：DP / 数组 | 难度：Hard | 标记：⭐

题目

EN: Each envelope [w, h]. One can fit into another if both dimensions are strictly smaller. Find max number nested.

中文：每个信封 [宽, 高]。一个信封能套到另一个里当且仅当严格小于。求最多能套几层。

澄清问题

  • EN: Can rotate? (No) / Equal dimensions count as fit? (No, strict) / Duplicates allowed?
  • 中：能旋转吗？相等算 fit 吗（严格小于）？重复信封？

🧠 整体思路

降维成 LIS（最长递增子序列）：

  1. 排序 trick：按 w 升序，w 相同的按 h 降序。
  2. 排序后，只需对 h 跑 LIS — 因为 w 相同时 h 降序保证它们不会互相套（严格小于不满足）。

LIS O(n log n)：patience sort 用二分。

复杂度：O(n log n)。

🪜 思路拆解

Step 1 · 排序 with tie-breaker - key = (w, -h)：w 升，h 降（用于解决严格小于）。

Step 2 · 对 h 跑 LIS O(n log n) - 维护数组 tails，tails[i] 是长度 i+1 的 LIS 的最小末尾。 - 遍历每个 h：用二分找第一个 ≥ h 的位置，替换或追加。

Step 3 · 返回 len(tails)

⌨️ 逐步实现

Step 1 · 排序

from bisect import bisect_left

def maxEnvelopes(envelopes):
    envelopes.sort(key=lambda e: (e[0], -e[1]))

Step 2 · LIS on h

    tails = []
    for _, h in envelopes:
        idx = bisect_left(tails, h)
        if idx == len(tails):
            tails.append(h)
        else:
            tails[idx] = h
    return len(tails)

Python 完整实现

from bisect import bisect_left

def maxEnvelopes(envelopes):
    envelopes.sort(key=lambda e: (e[0], -e[1]))
    tails = []
    for _, h in envelopes:
        idx = bisect_left(tails, h)
        if idx == len(tails):
            tails.append(h)
        else:
            tails[idx] = h
    return len(tails)

讲解策略

  1. 关键洞察 — 排序 trick：w 同时 h 降序，避免 w 相等时套娃。
  2. 转化成 LIS 是核心。
  3. patience sort + 二分 = O(n log n) LIS。
  4. 单独讲为什么 h 降序：举例 [3,3],[3,4]，如果都升序会被错误地套进 LIS。

常见坑

  • w 同时 h 升序：会错把 [3,3],[3,4] 当成可套，但 w 相等不严格 < 是不行的。
  • bisect_left vs bisect_right：严格 < 用 bisect_left，"<=" 用 bisect_right。
  • 二维 LIS 直接 O(n²) DP 也能过小数据，但 Hard 题最好给 O(n log n)。
