Google 原创 G04. Max Subarray Sum with Equal Endpoints 最大子段和 a[i]==a[j]

类别：📐 DP / 数组 · 难度：Hard · 标记：🆕 Google 原创

📐 DP / 数组 Hard 🆕 Google 原创

题目概览

🏷️ 类别：DP / 数组 | 难度：Hard | 标记：Google 原创 | 来源：NG R2 R2

题目

EN: Given array a, find (i, j) with i ≤ j and a[i] == a[j] that maximizes a[i] + a[i+1] + ... + a[j]. Return the (i, j) pair. Follow-up: O(1) extra space.

中文：数组 a，找 (i, j) 满足 i ≤ j 且 a[i] == a[j]，使得 sum(a[i..j]) 最大。返回 (i, j)。Follow-up：O(1) 额外空间。

澄清问题

  • EN: i == j allowed (single element)? / Negative numbers possible? / Multiple max — any one?
  • 中：i == j（单点）算吗？数组有负数吗？多个相同最大值返回哪个？

🧠 整体思路

核心：对每个 value v，找它在数组中最左和最右的位置 (l, r)。然后 max sum subarray with endpoints == v 就是 prefix[r+1] - prefix[l]。

遍历所有 unique v，取 max。

复杂度：O(n) — 一遍扫记录 first/last，再一遍算 prefix。

Follow-up O(1) space：用两次遍历： - 第一次找全局唯一值（首尾位置一致的）... - 实际上要 O(1) 较难，需要排序或更多前提。

🪜 思路拆解

Step 1 · 算 prefix sum - pref[i] = a[0] + a[1] + ... + a[i-1]。 - sum(a[i..j]) = pref[j+1] - pref[i]。

Step 2 · 对每个 value 记录 first / last 位置 - 一遍扫，HashMap：value → (first_idx, last_idx)。

Step 3 · 对每个 value 算 sum - sum = pref[last+1] - pref[first]，与全局 max 比较。

Step 4 · 返回最优 (i, j)

⌨️ 逐步实现

Step 1-2 · 前缀和 + first/last map

def max_sum_equal_endpoints(a):
    n = len(a)
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]

    first = {}
    last = {}
    for i, v in enumerate(a):
        if v not in first:
            first[v] = i
        last[v] = i

Step 3-4 · 取最大

    best_sum = float('-inf')
    best_i, best_j = -1, -1
    for v in first:
        i, j = first[v], last[v]
        s = pref[j + 1] - pref[i]
        if s > best_sum:
            best_sum = s
            best_i, best_j = i, j
    return best_i, best_j

Python 完整实现

def max_sum_equal_endpoints(a):
    n = len(a)
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]
    first, last = {}, {}
    for i, v in enumerate(a):
        if v not in first:
            first[v] = i
        last[v] = i
    best_sum, best_i, best_j = float('-inf'), -1, -1
    for v in first:
        i, j = first[v], last[v]
        s = pref[j + 1] - pref[i]
        if s > best_sum:
            best_sum, best_i, best_j = s, i, j
    return best_i, best_j

讲解策略

  1. 关键洞察：对每个 v，最佳 (i, j) 就是首尾位置（中间元素都已包含）。
  2. 前缀和 O(1) 查询区间 sum。
  3. 两遍扫，O(n) 时间 O(n) 空间。
  4. Follow-up O(1) space：诚实承认 hard，可以提"如果数组排序就可以"。

常见坑

  • 不是找最大 v：是找让 sum(a[i..j]) 最大的 v。负数可能让长 segment 反而和小。
  • first/last 同一位置：单点 (i=j) 也算合法，sum = a[i]。
  • 空数组：return (-1, -1) 或抛异常，先问。
