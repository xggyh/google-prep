16. 树中两点距离（LC 1740 思想）

来源：Cloud SDE Coding 1 · LC: LC 1740 ✅ 高置信 · 难度: Med

BFS LCA Tree ✅ 高置信 Med

题目概览

来源：Google Cloud SDE Coding 1 原描述：给 tree edges 和两个 node，问两个 node 相距多远。澄清了：edges 是否有重复、是否构成树、node 是否存在、是否同一 node。

题目

EN: Given tree edges as list of [u, v] (assume unweighted, undirected tree) and two nodes a, b. Return the distance between a and b.

中文：树的边列表（无权、无向、连通），给两个节点 a 和 b，返回它们之间的距离（边数）。

澄清问题（面经里的范本）

  • EN: Can edges contain duplicates? / Is it guaranteed to form a tree (connected + no cycle)? / Do a and b always exist in the tree? / What if a == b? / Tree size?
  • 中：边会有重复吗？保证是树吗（连通 + 无环）？a 和 b 一定存在吗？a 等于 b 怎么办？树的规模？

Python 实现

from collections import defaultdict, deque


def tree_distance(edges, a, b):
    if a == b:
        return 0
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)

    # BFS from a
    visited = {a: 0}
    queue = deque([a])
    while queue:
        node = queue.popleft()
        if node == b:
            return visited[node]
        for nb in graph[node]:
            if nb not in visited:
                visited[nb] = visited[node] + 1
                queue.append(nb)
    return -1  # b 不可达（树保证不会，但 defensive）

进阶 follow-up（多次查询）

如果对同一棵树有多次查询：预处理 LCA，每次查询 O(log n)。

def preprocess_lca(edges, n, root=0):
    # 倍增 LCA
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    LOG = (n - 1).bit_length() + 1
    depth = [0] * n
    up = [[-1] * n for _ in range(LOG)]
    # BFS 建立 depth 和 parent
    visited = [False] * n
    queue = deque([root])
    visited[root] = True
    while queue:
        u = queue.popleft()
        for v in graph[u]:
            if not visited[v]:
                visited[v] = True
                depth[v] = depth[u] + 1
                up[0][v] = u
                queue.append(v)
    for k in range(1, LOG):
        for v in range(n):
            if up[k-1][v] != -1:
                up[k][v] = up[k-1][up[k-1][v]]
    return depth, up, LOG


def lca(a, b, depth, up, LOG):
    if depth[a] < depth[b]:
        a, b = b, a
    diff = depth[a] - depth[b]
    for k in range(LOG):
        if (diff >> k) & 1:
            a = up[k][a]
    if a == b:
        return a
    for k in range(LOG - 1, -1, -1):
        if up[k][a] != up[k][b]:
            a = up[k][a]
            b = up[k][b]
    return up[0][a]

距离 = depth[a] + depth[b] - 2 * depth[lca(a, b)]。

讲解策略

  1. 澄清问题这一步面经里被特别表扬：照搬上面那 5 个问题。
  2. 单查询用 BFS：O(V+E) = O(n)，清晰。
  3. 如果是 follow-up "多次查询"：抛出 LCA 倍增 — O(n log n) 预处理 + O(log n) per query。
  4. 如果是 Euler Tour + RMQ：O(n) 预处理 + O(1) per query（更高级，不必写）。
  5. 不要直接上 LCA：先单点 BFS 拿稳基础分。

常见坑

  • 没问"是否一定是树"：如果只是 DAG 或一般图，BFS 还是对的但要确认。
  • 边有重复：BFS 不受影响（visited 控制），但要说一下。
  • a == b：直接返回 0，提前 short-circuit。
  • 递归 DFS 在深树上爆栈：BFS 更稳。
