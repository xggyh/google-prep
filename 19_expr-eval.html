19. 嵌套表达式求值 add/sub（LC 224/772 变体）

来源：Onsite NG R1 · LC: LC 224/772 变体 🟡 变体 · 难度: Med

Stack Parser Recursion 🟡 变体 Med

题目概览

来源：Google onsite NG 被拒 原描述：calculate math expression string，例 add(1, sub(1, 0)) 返回 2。用 iterative 而不是 split。

题目

EN: Evaluate a nested function-call expression with add(a, b) and sub(a, b) (and possibly arbitrary nesting). Examples: add(1, sub(1, 0)) → 2, add(add(1, 2), 3) → 6.

中文：求嵌套表达式的值，支持 add(a, b) 和 sub(a, b)，参数可以是数字或嵌套调用。

澄清问题

  • EN: Are arguments always exactly 2? Or variadic (add(1, 2, 3))? / Spaces allowed in input? / Negative numbers / multi-digit? / Only add/sub or more operations? / Whitespace handling? / Malformed input — throw or return error?
  • 中：每个函数总是 2 个参数还是变长？数字可以多位 / 负数 / 浮点？还有其他操作（mul/div）吗？空格如何处理？非法输入怎么办？

Python 实现（栈，迭代解析）

def evaluate(expr):
    expr = expr.replace(" ", "")
    stack = []   # 待处理的"操作 + 已收集的参数"
    num = ""
    op = None
    args = []
    i = 0

    while i < len(expr):
        ch = expr[i]
        if ch.isalpha():
            # 读操作名
            j = i
            while j < len(expr) and expr[j].isalpha():
                j += 1
            op_name = expr[i:j]
            # 期待下一个是 '('
            assert expr[j] == '('
            stack.append((op_name, []))  # 入栈：当前在收集这个 op 的参数
            i = j + 1
        elif ch.isdigit() or (ch == '-' and (i == 0 or expr[i-1] in '(,')):
            j = i + 1
            while j < len(expr) and (expr[j].isdigit()):
                j += 1
            stack[-1][1].append(int(expr[i:j]))
            i = j
        elif ch == ',':
            i += 1
        elif ch == ')':
            op_name, args = stack.pop()
            if op_name == "add":
                val = sum(args)
            elif op_name == "sub":
                val = args[0] - sum(args[1:])
            else:
                raise ValueError(f"Unknown op {op_name}")
            if stack:
                stack[-1][1].append(val)
            else:
                return val
            i += 1
        else:
            i += 1
    raise ValueError("Malformed expression")

讲解策略

  1. 画一棵 expression tree：让面试官看到你想成 AST。
  2. 栈解析 vs 递归下降：两种思路都能 work。
  3. 明确 token 类型：function name / ( / number / , / )。
  4. 每读到 ) 就 pop 一个 frame 求值：这是核心 invariant。
  5. dry run add(1, sub(1, 0))：栈状态一步一步写出来。
  6. 面经原话警告："Issues with minor syntax issue, out of bound error" — 这题是被拒的主因。面试时一定要慢、仔细、写完 dry run。

常见坑（这题挂了一个 candidate，注意学习）

  • out of bound：循环里 expr[j] 没判 j < len(expr)，越界。
  • 负数处理：sub(0, 1) 结果是 -1，参数也可能是负数 — 要在数字读取时判断 -。
  • 多位数：add(12, 34)，不能只读一位。
  • 空格：先 strip 是最稳的。
  • split 实现的坑：原帖说"没用 split"，因为 add(1, sub(1, 0)) 用 , split 会拆错。
