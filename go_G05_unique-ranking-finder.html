Google 原创 G05. Unique Ranking Finder 排名唯一可确定的选手

类别：🗺️ 图 / BFS / DFS · 难度：Med · 标记：🆕 Google 原创

🗺️ 图 / BFS / DFS Med 🆕 Google 原创

题目概览

🏷️ 类别：图 / DFS | 难度：Medium | 标记：Google 原创 | 来源：🐶ng R1 b2b

题目

EN: n players with unique true ranks 1..n. Higher-rank always beats lower-rank. Given m match results (a beat b), find all players whose rank can be uniquely determined, and output their ranks.

中文：n 个选手真实排名 1..n（高名次稳赢低名次）。给 m 场结果 (a 赢 b)。找出排名能唯一确定的所有选手 + 他们的具体排名。

澄清问题

  • EN: Can match results contradict (transitively)? / Multiple matches between same pair?
  • 中：比赛结果可能矛盾吗（A>B, B>C, 但 C>A）？同一对可以多次比？

🧠 整体思路

关键洞察：玩家 p 的排名可确定 ⟺ 我们能确定所有其他 n-1 个玩家都比他强或都比他弱。

形式化：对每个 p，定义： - stronger[p] = 比 p 强的玩家数（沿正图 p 能"输给"的传递关系）。 - weaker[p] = 比 p 弱的玩家数。

如果 stronger[p] + weaker[p] == n - 1，p 的排名唯一确定 = stronger[p] + 1。

实现： - 原图：p → 它输给的人（被强者打败）= 反向"a beat b"，即 b → a。 - 对每个节点跑两次 DFS：正图找所有能到达的（这些都比 p 强）；反图找所有能到达的（这些都比 p 弱）。

复杂度：O(n · (n + m))。

🪜 思路拆解

Step 1 · 建正图和反图 - 正图：graph[a].append(b) 表示 a 比 b 强（a beat b 时 a 更强）。 - 等价：从 b 出发能到达的 = b 弱于谁。 - 反图：rev[b].append(a) 表示 b 弱于 a。 - 从 a 出发能到达的 = a 强于谁。

实际上更直接的建图方式： - "a beat b" → 在"a 强于 b 的关系图"中加边 a → b（a 指向比它弱的）。 - 那么从 p 出发 DFS = p 强于谁的集合 → weaker[p]。 - 从 p 在反图出发 DFS = p 弱于谁的集合 → stronger[p]。

Step 2 · 对每个 p 跑 2 次 DFS 统计

Step 3 · 判断 + 返回排名 - 若 stronger[p] + weaker[p] == n - 1：排名 = stronger[p] + 1。

⌨️ 逐步实现

def unique_ranks(n, matches):
    forward = defaultdict(set)    # a -> {b 们}：a 强于 b
    reverse = defaultdict(set)    # b -> {a 们}：b 弱于 a
    for a, b in matches:
        forward[a].add(b)
        reverse[b].add(a)

Python 完整实现

from collections import defaultdict

def unique_ranks(n, matches):
    forward = defaultdict(set)
    reverse = defaultdict(set)
    for a, b in matches:
        forward[a].add(b)
        reverse[b].add(a)

    def reachable(start, graph):
        visited = set()
        stack = [start]
        while stack:
            u = stack.pop()
            for v in graph[u]:
                if v not in visited:
                    visited.add(v)
                    stack.append(v)
        return visited

    result = {}
    for p in range(1, n + 1):
        weaker_set = reachable(p, forward)
        stronger_set = reachable(p, reverse)
        if len(weaker_set) + len(stronger_set) == n - 1:
            result[p] = len(stronger_set) + 1
    return result

讲解策略

  1. 关键洞察：stronger + weaker == n-1 表示 p 和所有其他人都有可达关系。
  2. 正反两个图分别 DFS。
  3. 复杂度 O(n · (n + m))，n ≤ 1000 完全可以。
  4. 面经原帖说"被面试官 hint 才想出来" — 这个洞察确实需要点子。如果一开始没想到，先 brute force 枚举所有可能排列。

常见坑

  • 不要忘记 p 自己：visited 里不包含 p（DFS 起点）。
  • 重边：图用 set 去重。
  • 矛盾的输入：题目通常保证不矛盾，但要主动问。
  • rank = stronger + 1：因为有 stronger[p] 个人比 p 强，p 排在 (stronger + 1) 名。
