22. Delete Node in BST（LC 450）

来源：🐶家挂经 · LC: LC 450 ✅ 高置信 · 难度: Med

BST Tree Recursion ✅ 高置信 Med

题目概览

来源：🐶家挂经 原描述："题号45️⃣零" = LC 450，原帖说"没咋练结果来个树结构所以觉得是 hard"。

题目

EN: Given the root of a BST and a key, delete the node with that key. Return the modified BST root.

中文：BST 中删除值为 key 的节点，返回新的 root。

澄清问题

  • EN: Is BST guaranteed to have unique values? / What if key doesn't exist — return unchanged? / Need to maintain BST property after delete (yes obvious)? / Should we return reference to the original modified tree, or a new tree?
  • 中：BST 中值是否唯一？key 不存在时返回什么？需要返回原 root 还是新 root（修改原树 vs 不可变）？空树？

Python 实现

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


def deleteNode(root, key):
    if not root:
        return None
    if key < root.val:
        root.left = deleteNode(root.left, key)
    elif key > root.val:
        root.right = deleteNode(root.right, key)
    else:
        # 找到了要删的节点
        if not root.left:
            return root.right
        if not root.right:
            return root.left
        # 两个子节点：找右子树最小值代替
        succ = root.right
        while succ.left:
            succ = succ.left
        root.val = succ.val
        root.right = deleteNode(root.right, succ.val)
    return root

讲解策略

  1. 3 种情况分类清晰讲出来： - 叶子：直接删。 - 一个子节点：用子节点替换。 - 两个子节点：用右子树最小值（或左子树最大值）的值替换 + 递归删那个 successor。
  2. 画图走一遍：5 节点的 BST，删 root，让面试官跟。
  3. 复杂度：O(h)，平衡 BST 是 O(log n)，最坏 O(n)。
  4. 可以提 Morris 风格的 O(1) 空间，但不必写。
  5. Follow-up 可能问：保持 AVL/Red-Black 平衡？那是另一题。

常见坑

  • 找 successor 的位置：右子树最左节点（不是右子树最小的间接路径）。
  • 替换后忘记递归删：把 successor 的值复制到 root 后，要在右子树里把 successor 真的删掉。
  • 空树/key 不存在：return root 就行，但要主动覆盖。
  • 修改原树 vs 返回新树：题目通常允许 in-place，但要确认。
