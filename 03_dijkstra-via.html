03. Dijkstra 经过指定点（LC 743 / LC 1928 变体）

来源：L4 SWE AI/ML R1 · LC: LC 743/1928 变体 🟡 变体 · 难度: Med

Dijkstra Graph Heap 🟡 变体 Med

题目概览

来源：Google L4 SWE AI/ML 全套过经 R1 原描述：给一个 graph，每条边有距离，求出发点到目标点的最短距离。Follow-up：途中必须经过第三个点。

题目

EN: Given a weighted directed graph, source s, destination t. Return shortest distance from s to t, or -1 if unreachable. Follow-up: The path must pass through a given node m. Compute shortest distance through m.

中文：加权有向图，求 s→t 的最短路径长度。Follow-up：要求路径必须经过节点 m。

澄清问题

  • EN: Are weights non-negative? / Directed or undirected? / Can edges be zero or negative? / Are there duplicate edges? / What's the graph representation? / For follow-up: does "pass through m" mean visit m at least once (path can revisit nodes)?
  • 中：边权是否非负（决定能否用 Dijkstra）？有向还是无向？是否有重边？图怎么表示（邻接表/邻接矩阵/边列表）？Follow-up：经过 m 是要求路径上出现 m，还是允许重复访问节点？

Python 实现

import heapq
from collections import defaultdict


def dijkstra(graph, source, n):
    """graph: dict[node] -> list of (neighbor, weight)"""
    dist = {i: float('inf') for i in range(n)}
    dist[source] = 0
    pq = [(0, source)]
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]:
            continue
        for v, w in graph[u]:
            if d + w < dist[v]:
                dist[v] = d + w
                heapq.heappush(pq, (dist[v], v))
    return dist


def shortest(graph, s, t, n):
    d = dijkstra(graph, s, n)
    return d[t] if d[t] != float('inf') else -1


def shortest_via(graph, s, t, m, n):
    rev = defaultdict(list)
    for u in graph:
        for v, w in graph[u]:
            rev[v].append((u, w))
    d_from_s = dijkstra(graph, s, n)
    d_to_t = dijkstra(rev, t, n)
    if d_from_s[m] == float('inf') or d_to_t[m] == float('inf'):
        return -1
    return d_from_s[m] + d_to_t[m]

讲解策略

  1. 先确认权重非负，再讲 Dijkstra；如果可能有负边，要说"会改用 Bellman-Ford，O(VE)"。
  2. 手画一个例子：用 3-5 个节点的小图证明算法正确。
  3. Heap 优化：从 O(V²) 朴素 Dijkstra 升级到 O((V+E) log V)。
  4. Follow-up 关键点：shortest(s→t through m) = shortest(s→m) + shortest(m→t)。有向图要在反图上从 t 跑 Dijkstra 得 m→t。无向图直接两次正向 Dijkstra。
  5. 进一步 follow-up 可主动抛：「如果有多个必经点 m1, m2, ..., mk？」答：枚举顺序 + 多次 Dijkstra（k! 排列，k 小时可行；k 大就要 DP on subsets）。

常见坑

  • if d > dist[u]: continue 漏掉：会把过期节点重新处理，复杂度退化。
  • 无向图忘记反向加边。
  • Follow-up 在有向图里用正图算 m→t：错。要么反图，要么从 m 出发跑一次 Dijkstra。
  • float('inf') 加法溢出：Python 没问题，C++/Java 要用 LLONG_MAX/2。
