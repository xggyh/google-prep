07. N×N 网格放花（N-Queens 变种）

来源：L4 两轮 Onsite R1 · LC: 无直接 LC ❓ 无对应 · 难度: Hard

DFS Backtrack N-Queens ❓ 无对应 Hard

题目概览

来源：Google L4 两轮 Onsite R1 原描述：N×N 网格，'H' 是房子，'0' 是空地。放花 'F' 满足：(1) 每行每列恰好一朵 F；(2) 每个 H 四周恰好一朵 F。返回任意方案。

题目

EN: Given an N×N grid with 'H' (house) and '0' (empty). Place 'F' (flower) such that: 1. Each row and each column contains exactly one F. 2. Each H has exactly one F in its 4-neighborhood (up/down/left/right). Return any valid placement, or report infeasibility.

中文：N×N 网格，H=房子，0=空地。放 F=花，满足每行每列恰好一朵 F，每个 H 上下左右恰好有一朵 F。返回任意方案。

澄清问题

  • EN: Can F replace an H? Or only go on '0' cells? / Does "adjacent" include diagonals? / Is it guaranteed a solution exists? / N range? / If multiple H share an F, is that allowed (one F counts for two Hs)?
  • 中：F 能放在 H 上吗，还是只能放 '0'？四周是 4 邻还是 8 邻？是否保证有解？N 的范围？一朵 F 能同时被两个 H "声明"吗？

Python 实现

def place_flowers(grid):
    n = len(grid)
    cols_used = [False] * n
    result = [row[:] for row in grid]
    placements = []  # (row, col)

    def adjacent_count(r, c):
        cnt = 0
        for dr, dc in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
            nr, nc = r + dr, c + dc
            if 0 <= nr < n and 0 <= nc < n and result[nr][nc] == 'F':
                cnt += 1
        return cnt

    def valid_after_placement(r, c):
        # 每个已检查的 H 的邻居 F 数不能超过 1
        for hr in range(r + 1):  # 当前行及之前的行
            for hc in range(n):
                if grid[hr][hc] == 'H' and adjacent_count(hr, hc) > 1:
                    return False
        # 已完成的行（< r）的 H 必须恰好有 1 朵 F
        for hr in range(r):
            for hc in range(n):
                if grid[hr][hc] == 'H' and adjacent_count(hr, hc) != 1:
                    return False
        return True

    def dfs(row):
        if row == n:
            # 最后检查最后一行的 H
            for c in range(n):
                if grid[n - 1][c] == 'H' and adjacent_count(n - 1, c) != 1:
                    return False
            return True
        for col in range(n):
            if cols_used[col] or grid[row][col] == 'H':
                continue
            result[row][col] = 'F'
            cols_used[col] = True
            placements.append((row, col))
            if valid_after_placement(row) and dfs(row + 1):
                return True
            result[row][col] = grid[row][col]
            cols_used[col] = False
            placements.pop()
        return False

    return result if dfs(0) else None

讲解策略

  1. 类比 N-Queens：让面试官立刻知道你抓到了核心 — DFS + backtrack，按行枚举。
  2. 剪枝是关键（L4 帖子评论里有人专门问剪枝）： - 列已用：跳过。 - 当前格子是 H：跳过（F 必须放空地，前提是这样问过）。 - 放完当前行后，对所有可判定的 H（即上方所有邻居都已确定的 H — 也就是 row-1 行及更上面的 H）检查 F 邻居数是否合法。
  3. 复杂度：最坏 O(N!) ，但剪枝后实际可接受 N≤12 左右。
  4. 明确"不剪枝就 TLE"：N=8 没剪枝大约 8!=40320，加上 H 约束很快爆炸。
  5. 可选讨论：转成 SAT / ILP 在 N 很大时更稳，但面试场景 backtracking 足够。

常见坑

  • 没考虑 F 是否能放在 H 上：必须问清楚，影响搜索空间。
  • 剪枝时机错：等所有行都放完才检查 H 邻居数，等于没剪枝。
  • 每个 H "恰好 1 朵"理解偏差：注意是 == 1 不是 ≥ 1，过早 prune 不行。
  • 状态恢复 (backtrack) 忘记：grid、cols_used、placements 三个都要回滚。
